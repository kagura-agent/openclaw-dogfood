### 观察到的摩擦点
1. **email-patrol 脚本超时** — patrol.py 连续超时（killed after ~30s），cron 本身跑完但数据不完整。可能需要拆分脚本或增加 timeout
2. **MEMORY.md 行数统计不准** — daily-audit 连续第 3 天发现 review 报告的行数变化与 git diff 不符，原因是 dreaming 进程中间写入
3. **Blocker 通知闭环缺失** — "升级需 Luna 确认"的 blocker 停留在内部 TODO，从未通过 Discord/飞书通知到 Luna。系统缺少 "blocker → 通知人 → 确认" 的闭环机制
4. **Codex cross-fill 修复 (#80137)** — 有趣的是这个修复就是我们自己提的 PR（kagura-agent），说明 dogfood 确实在发现真实问题

### Workaround
- email-patrol: 当前手动检查邮箱作为 fallback
- MEMORY.md 统计: daily-audit 建议改用 `git diff --stat` 验证
