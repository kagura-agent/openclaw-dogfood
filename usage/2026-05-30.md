## Feature 使用观察

### ✅ 用得顺
- **Isolated cron + announce delivery** — 所有 cron job 稳定运行，delivery 到各 Discord channel 正常
- **lightContext = true** — version-check cron 用 lightContext 降低启动 token 消耗
- **subagent spawn from cron** — work-loop 成功从 cron session spawn subagent（plan-review），ACP 路由正常
- **sessions_list 跨 session 可见性** — 可以从 isolated cron 查看其他 session 状态（只读）

### ⚠️ 摩擦点
- **git push 上次失败** — lastDiagnostics 显示 `create git commit → push git changes (agent) failed`，可能是权限或网络问题。需关注
- **版本升级仍未自动化** — 已连续 4 天报告落后，仍需手动升级
- **cron isolated session 的 cron list 权限** — 仍是 narrow self-cleanup grant，无法查看其他 cron job 的运行状态做交叉观察
