## 🆕 今日新观察

### 1. cove session 1M context overflow + streaming dispatch transcript 损坏
**最严重的发现**。Kagura 自己描述："session 已重置干净（1M tokens 撑爆了 + 流式 dispatch bug 把 transcript 写坏）"。这是两个 bug 叠加：
- **bug a**: 1M token context 在长 session 中不可避免会触发，但应该有 graceful degradation（compaction / truncation），不该直接死
- **bug b**: streaming dispatch 在 overflow 时把 transcript 写坏 — 这是 data integrity 问题，应该上报 openclaw repo
**建议**: 写 openclaw issue 描述这个场景（session reset 行为 + transcript corruption）。

### 2. cove subagent 1.85M tokens 失控
单次 subagent 史上最高消耗。结合 `abortedLastRun=true, status=killed`，可能是：
- subagent 在 1M 上下文窗口内陷入循环（写测试 → 验证失败 → 重写）
- 没有及时 budget enforcement / token guardrail
**建议**: 给所有 subagent 默认加 timeoutSeconds + 上下文百分比 warning（如果有这个能力）。

### 3. community-ops token 累积失控（连续第三天）
