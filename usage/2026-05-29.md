## Feature 使用观察

### ✅ 用得顺
- **Isolated cron + announce delivery** — dogfood 日报稳定运行，delivery 到 Discord channel 正常
- **gh CLI changelog 获取** — 比 web_fetch 可靠，Cloudflare 不拦截
- **lightContext = true** — cron job 用 lightContext 降低启动 token 消耗，效果好

### ⚠️ 摩擦点
- **版本升级未自动化** — 每天报告新版本但需要手动升级。可以考虑加一个 upgrade cron 或 recommendation
- **Cron isolated session 无法读取其他 session 历史** — usage 观察步骤需要看其他 channel 的活动，但 isolated session 只有 narrow self-cleanup grant，无法 list 全部 cron jobs。需要改为 current session binding 或增加权限

### 📝 备注
- v2026.5.27 的 prompt-cache affinity 修复对我们 17+ cron jobs 有直接收益
- session write lock 超时释放修复值得关注，我们之前可能遇到过 session 卡住但归因不明的情况
