# OpenClaw v2026.5.26 Changelog (from v2026.5.22)

**Update available**: 2026.5.22 → 2026.5.26

## 🆕 对我们有用的新功能

1. **Cron 并发默认 8** — `cron.maxConcurrentRuns` 默认 8，我们不用手动配了，多个 cron job 可以并行跑
2. **Reaction 审批（Signal/iMessage/WhatsApp）** — 竖大拇指就能 approve，手机端不用打 `/approve` 了
3. **Activity Tab** — Control UI 新增 Activity 标签页，实时看 tool 活动摘要，调试 cron run 更方便
4. **TUI 排队提交** — 在 agent busy 时提交的 prompt 会排队而不是被丢弃
5. **Cron 相对时间 `+5m`** — 一次性 schedule 支持 `+5m` 这种写法，快速设提醒更方便
6. **Discord 模型选择分桶** — 模型列表超 25 时自动按字母分桶，不用翻页了
7. **QA coverage --match** — 可以按 query 筛选 proof scenarios，调试特定功能更快
8. **OpenAI sampling params 透传** — Gateway 现在透传 sampling params，可以精细控制生成参数

## 🔧 修复了我们遇到过的痛点

1. **Cron 恢复并发** — 之前 quota/circuit-breaker 暂停后恢复只有 1 并发，现在恢复到配置值（我们被坑过）
2. **sessions_send fallback 报错** — 之前 fallback 路由失败会给出模糊错误，现在报真实失败原因
3. **Codex context-window 溢出恢复** — 自动 compaction 重试，不再需要手动重启 session
4. **Discord 回复不再 echo bot 自己的消息到 prompt** — 用户 reply bot 消息时不再把 bot 之前的完整回复塞进 context
5. **Memory dreaming 聚焦** — REM dreaming 不再被旧 recall history 主导，fresh candidates 优先
6. **Commitment store 并发安全** — heartbeat 和 CLI 同时写不再丢状态了
7. **Telegram forum topic 名称传播** — topic 缓存正确传播，我们的 Telegram topic 管理更稳定

## ⚡ 值得关注但暂不采用的

1. **Rastermill 替代 Sharp** — 图片处理不再依赖 Sharp，理论上装包更快，但我们暂不需要改动
2. **Transcript 核心化** — 会议摘要、source-provider chunks 统一走 transcript path，未来做会议 bot 有用
3. **Voice SDK 统一** — realtime voice 路径统一 SDK，做 voice agent 时参考
4. **OpenTelemetry LLM spans** — 可观测性增强，等我们需要 tracing 时再开

## 升级建议

建议尽快升级，主要动力：
- Cron 并发默认 8 + 恢复修复 = 我们的 cron 稳定性直接提升
- Reaction 审批对手机端体验提升大
- sessions_send 修复对 multi-agent 协作很关键
