## 🆕 对我们有用的新功能（能立即采用）

### 1. 原生 /usage 完整 footer 渲染器 + 默认模板 (#92657, #89835, #89629)
内置 full footer 渲染、per-turn usageState hook、credential-aware limits、fixed-decimal 格式化。
**对我们有用**: 我们的 cron 任务消耗大量 token，有了原生 usage footer 可以在 Discord 投递中直接看到每次 run 的消耗，不用再手动跟踪 token 使用趋势。

### 2. /btw 支持 CLI-backed sessions (#92669)
CLI-backed session 现在支持 `/btw` 命令。
**对我们有用**: 可以在 ACP session 运行中异步发送补充信息，适合我们 code-review 和 work-loop 中需要中途追加上下文的场景。

### 3. Session identity 暴露到 runtime prompt (#92468)
session identity 现在出现在 runtime prompt 中，agent 能感知自己在哪个 session。
**对我们有用**: 我们的 multi-channel 架构中 agent 经常需要知道当前在哪个 channel session 运行，这个信息之前需要手动传递。

### 4. Browser --labels overlay 扩展到 full-page 和 element captures (#92834)
浏览器截图标注现在支持全页面和单元素模式。
**对我们有用**: browser-automation skill 截图分析更精确，适合网页操作场景。

## 🔧 修复了我们遇到过的痛点

### 5. Yielded subagent runs 在 abort 时正确 pause (#92631)
修复: restart 时 yielded 的 subagent 不再丢失。
**解决痛点**: 我们 06-16 观察到 cove session aborted，如果是 restart 导致的，这个修复能防止类似问题。

### 6. Reset archive fallback 恢复 (#92879)
修复: 当 active async transcript 不可用时，回退到 reset archive 读取。
**解决痛点**: 长期 channel session 的 context 恢复更可靠，适用于我们的 #cove 和 community-ops 等持久 session。

### 7. 主 session heartbeat 事件去重 (#91287)
修复: 主 session 不再收到重复 heartbeat systemEvent。
**解决痛点**: 我们有 10+ 个 cron，heartbeat 去重能减少不必要的 token 消耗。

### 8. Discord auto-thread title timeout + reasoning model output cap (#64734)
auto-thread title 现在有 60s 超时和 4096 token reasoning 输出上限。
**解决痛点**: Discord thread 创建偶尔卡住的问题应该能修复。

### 9. memory_search 在 transient QMD mode 下保持可用 (#92618, #92639)
修复: QMD backend 在 transient 模式下 memory_search 不再 disabled。
**解决痛点**: 确保所有 cron isolated session（本质是 transient）都能正常用 memory_search。

### 10. SQLite 避免 NFS 卷上使用 WAL (#91247)
修复: 状态存储在 NFS 挂载时不再因 WAL 冲突出错。
**基础设施**: 虽然我们不用 NFS，但提升了整体存储可靠性。
