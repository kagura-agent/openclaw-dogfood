# OpenClaw Changelog — 2026-05-16

## 版本变化
- **当前版本**: 2026.5.3-1
- **最新版本**: 2026.5.12
- **跨版本升级**: 9 个小版本差距，这是一个大版本跳跃

## 🆕 对我们有用的新功能（可立即采用）

### 1. `acp.fallbacks` — ACP 后端降级
当主 ACP 后端不可用时自动切换到备用后端。我们多 runtime 配置的场景（Claude + OpenAI）正好用得上，避免单点故障。

### 2. `cron.get` — 单个 cron 任务查看
可以通过 `cron.get <id>` 查看单个任务详情，调试 cron 时不用再 list 全部了。我们有 10+ cron 任务，这是刚需。

### 3. 每 sender 工具策略 (`tools.byProvider`, `tools.message.crossContext`)
可以按发送者身份限制危险工具。我们的 Discord 群开放给社区，这能精细控制谁能触发什么。

### 4. `/context map` — 上下文可视化
发送 treemap 图片展示当前 session context 各成分占比。调试 token 爆炸问题的利器。

### 5. Subagent model 默认值修复 (#81395)
`agents.defaults.subagents.model` 现在正确优先于 agent 自身模型。我们的 subagent 模型经常需要不同于 parent，这修复了一个潜在的配置陷阱。

### 6. Telegram HTML 格式保留
cron 定时发送的消息现在保留 Markdown/HTML 格式（链接可点击）。我们的晨报、财报等 cron 输出直接受益。

### 7. Control UI auto-scroll 模式选择器
可以选择"始终跟随"/"接近底部才跟随"/"手动"。长输出 session 监控时不再自动跳走。

## 🔧 修复了我们遇到过的痛点

### 8. Telegram 流式回复不再重复前缀文本 (#78946)
工具调用循环中的文本不再累积重复。我们在 Telegram 上看到过这个问题——回复越来越长，前面的段落重复出现。

### 9. CLI 未知子命令不再误导到 plugins.allow (#80123)
修复了 `openclaw xxx` 时错误建议 "add to plugins.allow" 的问题。**这个 fix 就是我们 PR #82128 的上游合并！** 🎉

### 10. Cron timeout 修复 (#79426)
cron payload 里的 `timeoutSeconds` 现在被正确尊重，不再被全局 120s 默认值覆盖。我们的长时间 work-loop cron 经常需要更长 timeout。

### 11. Subagent 完成时不再触发 parent 的多余 LLM 调用 (#66749)
修复了 subagent exec 完成后 parent session 被无谓唤醒的问题。节省 token 消耗。

### 12. 死循环工具调用检测 (#80668)
相同工具调用重复返回相同结果时自动中止。我们偶尔见过 agent 陷入 read→read→read 循环。

## ⚡ 值得关注但暂不采用的

### 13. Telegram 隔离 worker 轮询 (#81746)
Bot API 轮询移到独立 worker + 本地持久化 spool。大幅提升 Telegram 通道稳定性，但需要升级后观察。

### 14. WhatsApp 外部化为 ClawHub 插件
WhatsApp 不再是核心依赖。我们暂未用 WhatsApp，但架构方向值得关注——更轻量的核心。

### 15. Slack unfurlLinks/unfurlMedia 配置
Slack 链接/媒体预览可控。我们暂未用 Slack，记录备用。

### 16. 大量安全加固
Gateway、sandbox、browser、node pairing 全面安全审查。升级时自动获得，无需手动配置。
