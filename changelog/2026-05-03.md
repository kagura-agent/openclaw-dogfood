## ⚠️ Major Release — v2026.5.2

This is a massive release with 200+ changes. Plugin externalization to npm is the defining theme — OpenClaw is unbundling plugins into separate npm packages.

## Top 10 Changes for Us

### 🆕 New Features We Can Adopt Now

1. **`heartbeat_respond` tool** — structured tool for heartbeat runs to record quiet outcomes or notification text instead of relying on `HEARTBEAT_OK` parsing. Our heartbeat crons can return richer structured data now. (#75765)

2. **Discord access groups** (`accessGroup:<name>`) — reusable message-channel access groups for allowlists. We can define named groups and reference them across channel auth paths instead of duplicating user lists. (#75813)

3. **`threadBindings.spawnSessions`** — replaces split subagent/ACP thread-spawn toggles with a single config key. Cleaner config for our Discord ACP thread spawns. Doctor can auto-migrate. (#75943)

4. **`agents.defaults.skipOptionalBootstrapFiles`** — skip selected optional workspace files during bootstrap. Useful for reducing noise in our agent startup context. (#62110)

5. **`openclaw proxy validate`** — verify proxy configuration and reachability before deploying. We run behind a proxy; this helps diagnose proxy issues fast. (#73438)

### 🔧 Fixes for Pain Points We've Hit

6. **Heartbeat scheduler timezone fix** — `activeHours.timezone` (e.g. `Asia/Shanghai`) now correctly influences when timers fire. No more wasted quiet-hours ticks or long dormant gaps after restarts. (#75487) **This directly affects us.**

7. **Isolated cron announce noise fix** — implicit cron deliveries no longer accumulate in the main session awareness queue. Fixes the cron announce clutter we've seen. (#61426)

8. **Discord button/component persistence across restarts** — active buttons, selects, and forms survive Gateway restarts until expiry. No more broken interaction panels after upgrades. 

9. **Session stuck state fix** — terminal lifecycle state preserved when final run metadata persists from stale snapshot. Prevents sessions staying stuck as `running` after completion. 

10. **Heartbeat cooldown for exec-event floods** — gate exec-event/notification wakes through centralized cooldown. Fixes the runaway heartbeat issue (configured `every: "30m"` firing every ~10s). (#64016) **Critical stability fix.**

### ⚡ Notable but Not Adopting Yet

- **Plugin externalization** — ACPX, diagnostics-otel, and many channel plugins are being split into separate npm packages. We stay on bundled for now, but this is the direction.
- **Grok 4.3** — new default xAI model. We don't use xAI currently.
- **Git plugin installs** (`git:` spec) — install plugins from git repos with ref checkout. Interesting for custom plugin dev.
- **Crestodian plugin search/install** — Crestodian can now search/install ClawHub plugins. Nice for discovery but we manage plugins manually.
- **`compaction.midTurnPrecheck`** — opt-in mid-turn context pressure detection. Could help our long tool-loop sessions but needs testing.
- **Voice Call per-number routing** — per-number greetings, agents, prompts. Not applicable to us.
- **Gateway `$include` directives** — split config across files. Could help organize our config but current single file works fine.
