# OpenClaw Changelog - 2026-05-04

## Version Update: 2026.4.26 → 2026.5.2

Major release with 200+ changes. Key highlights for our setup:

### 🆕 New Features We Can Adopt Immediately

1. **`heartbeat_respond` tool** — structured tool for heartbeat runs to record quiet outcomes or explicit notifications without relying on `HEARTBEAT_OK` parsing. We can adopt this for cleaner heartbeat cron handling.

2. **`threadBindings.spawnSessions` unified config** — replaces split subagent/ACP thread-spawn toggles. Simplifies our Discord thread ACP setup. `openclaw doctor --fix` migrates legacy keys.

3. **Discord access groups (`accessGroup:<name>`)** — reusable message-channel access groups for DM authorization. We can consolidate our allowlist config.

4. **`agents.defaults.skipOptionalBootstrapFiles`** — skip selected optional workspace files during bootstrap. Useful for faster subagent spawns.

5. **Heartbeat scheduler active-hours-aware** — scheduler now seeks forward to first in-window phase slot instead of arming timers for quiet-hours. Non-UTC `activeHours.timezone` (like our `Asia/Shanghai`) now correctly influences timer firing. Fixes wasted quiet-hours ticks.

6. **`openclaw proxy validate`** — verify proxy config, reachability, and allow/deny behavior before deploying. Useful for our proxy setup validation.

7. **Cron isolated announce stays out of main session** — isolated cron jobs no longer accumulate in main conversation awareness queue. Cleaner main session.

### 🔧 Fixes For Pain Points We've Hit

1. **Discord typing indicators alive during long tool runs** — no more idle-looking chat during extended agent work. Fixes #42652.

2. **Discord buttons/selects survive gateway restarts** — multi-step interactions less likely to break during upgrades. We've hit this.

3. **Session stuck states preserved** — terminal lifecycle state preserved correctly, preventing sessions stuck as "running" after completion/timeout.

4. **Gateway `sessions.list` responsive on large stores** — reuses cache/indexes instead of heavyweight summaries. We have many sessions.

5. **Heartbeat cooldown gate** — prevents runaway heartbeat runs from exec-event/notification/spawn/retry wakes that were self-feeding. Fixes the "30m fires every 10s" bug.

6. **Discord status reactions cleanup** — stale thinking emoji removed after run completion. We've seen permanent thinking emojis.

7. **ACP Discord duplicate suppression** — thread-bound ACP replies no longer delivered twice. Fixes #60780.

8. **Cron malformed job tolerance** — one bad cron entry no longer aborts the whole scheduler tick.

9. **Auto-reply group chat fallback** — Discord group turns no longer silently complete without visible reply when message tool unavailable. Fixes #74868 (we reported this!).

10. **Memory CRLF/duplicate marker self-heal** — Dreaming and Memory Wiki files self-heal from repeated generated sections.

### ⚡ Worth Noting But Not Adopting Yet

1. **Plugin externalization to npm** — ACPX, diagnostics-otel, and many channel plugins being externalized. We stay on bundled for now.

2. **Grok 4.3 default for xAI** — new model available if we want to try xAI.

3. **`git:` plugin installs** — install plugins from git refs. Interesting for custom plugin dev.

4. **Mid-turn compaction precheck** (`agents.defaults.compaction.midTurnPrecheck`) — detects tool-loop context pressure mid-turn. Opt-in, worth testing later.

5. **Voice Call per-number inbound routing** — not relevant to us yet but shows voice maturation.

6. **Codex app-server improvements** — isolated `CODEX_HOME` per agent, better crash recovery. Relevant when we use more Codex.

7. **Gateway performance improvements** — leaner hot paths across startup, sessions, plugin loading, prompt prep. Should feel faster without config changes.
