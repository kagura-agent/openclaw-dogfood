## 🆕 对我们立即有用的新功能

### 1. Plugin SDK: `defineToolPlugin` + `openclaw plugins build/validate/init`
一句话：可以用 TypeScript 写 typed tool plugin 了，自动生成 manifest。
**为什么有用**：我们一直想给 finance/trading 做自定义 tool plugin，现在有了正式 SDK 路径，不用再 hack MCP。

### 2. Skills CLI: `--global` flag for `openclaw skills install/update`
一句话：技能安装/更新可以指定 `--global` 管理共享技能。
**为什么有用**：多 agent 共享技能场景（比如 meme-maker、github）不用每个 agent 单独装了。

### 3. Browser modal dialog handling
一句话：browser snapshot 现在显示 pending/handled modal dialogs，新增 `browser dialog --dialog-id` 命令。
**为什么有用**：browser-automation 遇到 alert/confirm 弹窗不再卡住，自动化更稳健。

### 4. `browser evaluate --timeout-ms`
一句话：browser evaluate 可以自定义超时了。
**为什么有用**：之前跑长时间页面函数会超时，现在可以调大。

### 5. Cron: isolated task runs linked to stable cron session
一句话：定时任务的 isolated run 现在关联到稳定的 cron session。
**为什么有用**：cron task 的状态追踪和清理更可靠了，不会再丢 orphan runs。

## 🔧 修复了我们遇到过的痛点

### 1. Memory search: bounded batch scanning for large chunk tables
一句话：memory search 的 JS fallback path 不再阻塞 Node.js 主线程。
**为什么有用**：我们 memory 越来越大，之前偶尔搜索卡顿，这个修了根因。

### 2. Skills: refresh skill snapshots when watched skill roots change
一句话：改了技能目录后不用新开 session 就生效了。
**为什么有用**：之前改 skill 文件后要重启 session 才能用新版，很烦。

### 3. Discord: deliver final replies in progress-mode preview streams
一句话：Discord 进度预览模式下最终回复不再被吞掉。
**为什么有用**：我们 Discord 频道里偶尔看到回复"消失"，应该就是这个 bug。

### 4. Cron/Discord: scheduled agent turns post once instead of echoing
一句话：cron announce 不再重复发送。
**为什么有用**：之前某些 cron 输出会在频道里发两次。

### 5. Subagent completion: preserved keep-mode payloads after retry exhaustion
一句话：subagent 完成结果不再因 retry 耗尽而丢失。
**为什么有用**：我们大量使用 subagent，偶尔丢结果的问题有望解决。

## ⚡ 值得关注但暂不采用

### 1. Android realtime voice sessions via Gateway relay
一句话：Android Talk Mode 走实时 Gateway voice relay 了。
**关注原因**：我们目前没用移动端，但未来手机 agent 交互会很强。

### 2. Gateway startup latency reduction
一句话：启动时 plugin service 和 channel sidecar 并行启动。
**关注原因**：重启更快了，升级后体感会好。

### 3. Telegram forum topic isolation improvements
一句话：Telegram topic 之间不再互相阻塞。
**关注原因**：我们主要用 Discord，但 Telegram 场景的稳定性也在提升。

### 4. Config reload metadata: hot-reloadable vs restart-required
一句话：config 工具现在能告诉你改了某项是否需要重启。
**关注原因**：之前改 config 不确定要不要重启，这个很实用，等升级后自然享受。

### 5. GPT-5.5 vs Claude Opus 4.7 QA parity gates
一句话：QA-Lab 更新了 GPT-5.5 和 Opus 4.7 的 parity 对比基准。
**关注原因**：模型选择参考，但不需要我们主动做什么。
