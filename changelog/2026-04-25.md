## 2026.4.23 Highlights (NEW — not yet installed)

### Changes We Should Care About

1. **Image generation via Codex OAuth** — `gpt-image-2` works without `OPENAI_API_KEY`. We currently don't use image gen through OpenAI, but this removes a friction point.
2. **OpenRouter image generation** — Image models via `OPENROUTER_API_KEY`. Could expand our kagura-canvas options.
3. **`sessions_spawn` forked context** — Children can now inherit requester transcript. This is big for our subagent workflows — context-heavy delegations won't need manual context injection.
4. **Per-call `timeoutMs` for generation tools** — Image/video/music/TTS can have per-request timeouts. Useful for ComfyUI workflows that sometimes take long.
5. **Memory `contextSize` config** — `memorySearch.local.contextSize` tunable. We run local embeddings; this could help.
6. **Pi packages → 0.70.0** — GPT-5.5 catalog improvements.
7. **Codex harness debug logging** — Better visibility into harness selection decisions.

### Fixes We Might Hit

1. **Block streaming duplicate replies** — Fixed: partial block-delivery aborts no longer cause double messages. We've seen occasional duplicates — this might be the fix.
2. **Telegram media replies** — Markdown image syntax now properly becomes media in Telegram groups. We don't use Telegram heavily but good to know.
3. **Memory QMD recreation** — Stale QMD collections repair properly. Memory health improvement.
4. **MCP cleanup on session end** — Orphaned MCP processes no longer accumulate. Good for our 46-cron setup.
5. **Codex `request_user_input` routing** — Now routes back to originating chat. Important for ACP thread sessions.
6. **`openai-codex/gpt-5.5` synthesis** — Cron/subagent runs won't fail with `Unknown model` for Codex OAuth.

## 2026.4.22 (Current — installed since last check)

This is a massive release with 40+ changes and 50+ fixes. Key items for us:

### Already Benefiting From
