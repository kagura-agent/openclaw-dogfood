## 🆕 对我们有用的新功能（立即可采用）

### 1. Skill Workshop 全面上线
Skill Workshop 现在支持 pending proposals、CLI/Gateway review actions、rollback metadata、`skill_workshop` agent tool，以及 Control UI 全流程（提案列表、today 视图、revision 对话框、可搜索文件预览、session handoff）。
→ **我们的 skill 开发和审核流程可以从手动 git 管理迁移到 Skill Workshop 的 governed review flow**。

### 2. Workboard 编排原语
新增多 agent 规划和运行追踪的编排工具，支持 task-backed board runs 和 task comments。
→ **我们的 team-lead / flowforge 多 agent 工作流可以利用 Workboard 做更结构化的任务追踪**。

### 3. Cron 重试机制改进
Cron job 遇到临时性 model rate limit 后会自动重试，不再等到下一个 schedule slot。
→ **直接解决我们偶尔遇到的 cron job 因 rate limit 静默跳过的问题**。

### 4. MiniMax M3 模型支持
Provider 新增 MiniMax M3。
→ **多了一个可用的模型选择，可用于特定场景的 fallback**。

### 5. Chat UI 增量流式输出 + 首输出延迟追踪
Chat deltas 改为增量流式，跳过 streaming 时的 markdown parsing，追踪 first-output latency。
→ **Control UI 使用体验会明显改善，尤其是长回复时**。

## 🔧 修复了我们遇到过的痛点

### 6. Memory 系统稳定性大修
Memory 写入串行化、Linux watcher fan-out 减少、transient read 重试、目录重建后 watcher 自动重连、transcript 路径 rollover 重写。
→ **我们之前偶尔遇到的 memory search 结果不稳定 / watcher 丢失应该被修复了**。

### 7. Disabled skill SecretRef 不再中断 turn
禁用的 skill 的环境变量覆盖会被跳过，不再因为 stale apiKey SecretRef 导致 channel turn abort。
→ **之前偶尔遇到的莫名 turn 失败可能就是这个原因**。

### 8. 插件加载失败隔离
单个 bad plugin 不再 poison 其他插件的 runtime path，失败指引更清晰。
→ **插件管理更安全了，不用担心一个坏插件影响全局**。

### 9. Channel 超时保护增强
Telegram、Discord、WhatsApp、Signal、Feishu 等所有 channel 的 request/retry timer 都加了 cap。
→ **减少 channel 挂起的风险，我们的 Discord 集成更稳定**。

### 10. Agent 中断恢复改进
Agent 和 CLI runtime 在 interrupted tool calls、stale session bindings、compaction handoffs、media delivery retries 场景下恢复更干净。
→ **长时间运行的 subagent/ACP session 中断后恢复更可靠**。

## ⚡ 值得关注但暂不采用

- **Code mode 内部命名空间**: 为 scoped agent/global sessions 添加了命名空间和精确 tool dispatch，面向更复杂的多 agent 场景

- **Tokenjuice 插件外部化**: 独立为 `@openclaw/tokenjuice` 插件
