## 2026.7.2-beta.3 新特性摘要

### 🆕 对我们有用的新功能（可立即/近期采用）

1. **Skill Workshop 自动审批** — apply/reject/quarantine 不再需要额外 approval prompt，默认自动处理。我们大量使用 Workshop，这直接减少摩擦。
2. **Skill Workshop 历史回顾** — 自动扫描旧 session 寻找 skill 创建建议，存为 pending proposals。适合发现我们反复手动做的模式。
3. **Cron 运行历史** — `cron runs` 从 task ledger 提供历史记录。我们跑 15+ cron jobs，排查问题终于有历史可查了。
4. **Discord voice 参与者上下文** — agent 可以知道谁在语音频道。我们有 Discord voice 集成，这让 voice 场景更智能。
5. **Channel progress 改进** — 长时间运行的任务可以预留 progress draft，用模型 preamble 做标题。我们的 work-loop 经常跑 2+ 分钟，用户体验改善。
6. **MCP session 隔离** — MCP server 连接按 session 隔离。多 session 并行更安全。

### 🔧 修复了我们遇到过的痛点

7. **Cron lifecycle conflict 重试修复** — 防止 post-execution claim conflict 重放已完成的消息/工具。我们偶尔看到 cron 行为异常，这可能是根因。(#108428)
8. **Unicode 安全截断（巨大修复）** — 40+ PR 修复了 CJK/emoji/surrogate pair 在预览、提示、诊断、标签、session key 等各处的截断问题。**我们是重度 CJK 用户，这是最重要的修复之一。**
9. **Skills prompt compaction** — 确保所有 skill identity 在 budget 截断前保留。我们有 30+ skills，之前可能有 skill 在压缩时丢失。
10. **Cron 列表表格 CJK 对齐** — `cron list` 表格按终端显示宽度计算列宽，CJK/emoji 不再错位。(#103616)
11. **Web Readability 相对链接** — web_fetch 返回的文章链接现在正确解析。我们的 study-loop 和 research 大量依赖 web_fetch。

### ⚡ 值得关注但暂不采用的

12. **GPT-5.6 默认模型** — 新安装默认 `openai/gpt-5.6`。我们已有配置，不受影响，但可以测试 5.6 作为 fallback。
13. **Cloud Workers** — 远程 session 执行和 worker routing。目前我们单机运行，未来扩展时很有用。
14. **Remote coding sessions** — 在 cloud worker 上跑 Control UI session，终端 resume OpenCode/Pi。适合分布式开发场景。
15. **Linux deb/AppImage** — 新的安装打包格式。我们用 npm 安装，暂不需要。
16. **External supervisor mode** — `OPENCLAW_SUPERVISOR_MODE=external` for OCM。我们自管理，暂不需要。
