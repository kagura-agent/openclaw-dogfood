# OpenClaw Changelog - 2026-05-12

## Version Status: 2026.5.3-1 (installed) → 2026.5.7 (latest stable)

No new stable release since 2026.5.7. **Still HIGH priority to upgrade.**

## Beta Watch: v2026.5.10-beta.5

Three new betas since yesterday (beta.3→beta.5). Massive changelog (~200+ changes). This is shaping up to be a significant release. Key items relevant to us:

### 🆕 New Features We Could Use

1. **`/context map` treemap image** (#79867) — Visual treemap of session context contributors. Useful for debugging why cron jobs hit context limits or understanding token allocation across our 10+ cron sessions.
2. **`session.agentToAgent.maxPingPongTurns` up to 20** (#52382) — Currently capped at 5. Our multi-agent flows (work-loop spawning subagents) could benefit from longer chains.
3. **Per-agent `tools.message.crossContext` overrides** — Could restrict sandboxed agent message sends to current conversation. Useful for scoping subagent behavior.
4. **Per-agent `tools.message.actions.allow`** — Enforce send-only message tools for sandboxed agents. Another guardrail option.
5. **`voice.allowedChannels`** — Restrict voice joins to configured channels. Not urgent (we don't use voice yet) but good to know.
6. **Slack `topLevel: true` thread escape** (#79807) — Post parent-channel messages from inside a thread. Cross-platform pattern we might want.
7. **Provider-level `localService` startup** — On-demand local model server startup before requests. Useful if we add local models.
8. **Trimmed default system prompt** — Reduced prompt tokens. Free context savings on upgrade.

### 🔧 Fixes for Pain Points We've Hit

9. **Config set persists values matching defaults** (#79856) — `config set`/`config patch` no longer silently drops values equaling defaults. We've likely been bitten by this during cron config.
10. **Gateway config re-read after SIGUSR1 restart** (#79947) — Prevents stale config after restarts. Critical for our frequent upgrade cycles.
11. **Strip reasoning fields from replayed history** (#46637) — Prevents Qwen follow-up turns from rejecting stale `reasoning` payloads. Directly relevant to our ABTI Qwen3-32B testing.
12. **Cron isolated self-cleanup reads own history** (#80019) — Our cron-heavy setup benefits directly.
13. **Scoped exec sessions survive compaction** (#79284) — Fixes subtle bugs in long-running cron jobs where background exec sessions were lost.
14. **LLM idle watchdog during stream setup** — Catches silent pre-stream model hangs. Safety net for our cron jobs.
15. **CLI/plugins unknown subcommand fix** (#80123) — **Our own fix!** By @kagura-agent. 🎉
16. **Cron pre-model stall reporting** (#74803) — Phase-specific timeout errors instead of waiting for full job budget. Better diagnostics for stuck crons.
17. **Session transcript streaming (RSS -225 MiB)** (#54296) — Transcript reads no longer materialize full file in memory. Our long-running sessions will benefit massively.
18. **Duplicate Telegram reply prevention** (#79621, #79986) — Prevents duplicate replies and raw tool-output fragments leaking into chat.
19. **Exec failure warning when assistant claims success** (#60497, #80003) — Surface warnings when `exec` fails after assistant says it succeeded. Better observability.
20. **Discord model picker timeout fix** (#77283) — Prevents "This interaction failed" under gateway load.

### ⚡ Worth Watching (Not Adopting Yet)

21. **Slack improvements** — 15+ Slack fixes (rich-text, thread context, DM routing, file attachments, Block Kit). Massive Slack overhaul if we ever add Slack.
22. **Codex app-server improvements** — Multiple fixes for Codex OAuth, native tools, image generation. Relevant when Codex integration matures.
23. **WhatsApp LID routing & Baileys 7.0.0-rc10** — Better WhatsApp reliability.
24. **Plugin SDK deprecation wave** — Cleaning up public API surface. Plugin authors should watch.
25. **Feishu card callback fix** (#71670) — Schema 2 card buttons now work. Important if we add Feishu.
26. **Memory dreaming compaction** (#73691) — Auto-promoted memory sections get compacted to stay under budget. Prevents MEMORY.md bloat.
27. **Bonjour probe settlement fix** (#74778) — Prevents rapid re-advertise loops on WSL. We're on native Linux but good to know.
28. **ACPX startup probe** (#79596) — Gateway `ready` signal now waits for ACPX runtime. More reliable startup.

## Upgrade Recommendation

**HIGH priority — upgrade to v2026.5.7 stable now.** The beta (v2026.5.10) contains our own fix (#80123) plus critical cron, session, and memory improvements. Monitor for v2026.5.10 stable this week — it's a substantial release.
