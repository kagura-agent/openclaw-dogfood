# OpenClaw Changelog — 2026-05-25

- **当前版本**: 2026.5.20
- **最新版本**: 2026.5.22
- **状态**: ⚠️ 落后 2 个版本（v2026.5.21 跳过，直接到 v2026.5.22）
- **上次升级**: 2026-05-21 (升级到 2026.5.20)

## 🆕 对我们有用的新功能（可立即采用）

### 1. Meeting Notes 外部插件
新增 `meeting-notes` 插件，支持 Discord voice 实时会议记录捕获、手动 transcript 导入、`openclaw meeting-notes` CLI 访问。
→ **对我们有用**: Luna 跟 Kagura 在 Discord voice 交流时可自动记录，替代手动笔记。

### 2. Gateway 启动性能大幅优化（~4100× `/models` 加速）
- Provider auth-state 预热从 ~20s 降到 ~5ms
- 懒加载启动空闲的 plugin 工作、handler、ACPX runtime
- 缓存 plugin SDK alias maps，跳过无关的 macOS Linuxbrew PATH 探测
→ **对我们有用**: 重启 gateway 后恢复速度显著提升，`/models` 不再卡顿。

### 3. Chat Session Picker 支持搜索和分页
Control UI 的 session picker 新增搜索和 Load More 分页。
→ **对我们有用**: 我们有 39 个 cron + 多个 channel session，找历史 session 终于不用滚半天了。

### 4. Plugin SDK: embeddingProviders 能力合约
Embeddings 成为可复用的 provider surface，不再绑定 memory-specific adapter。
→ **对我们有用**: 未来自定义 memory/wiki 搜索可以用统一 embedding 接口。

### 5. Plugin SDK: Poll sender
通用 channel-message poll sender，channel 插件可以暴露投票功能。
→ **对我们有用**: Discord poll 功能更稳定，可用于社区互动。

## 🔧 修复了我们遇到过的痛点

### 6. ACP 子 session 孤儿进程泄漏修复
`sessions_spawn` 创建的 ACP 子 session 在 parent session reset/delete 时正确关闭，不再留下孤儿 `claude-agent-acp` 进程耗尽内存。
→ **直接影响**: 我们频繁用 ACP 跑 Claude Code，之前偶尔发现内存增长，可能就是这个原因。

### 7. Cron network retry 修复
`cron.retry.retryOn: ["network"]` 现在正确处理 `EAI_AGAIN`、`EHOSTUNREACH`、`ENETUNREACH`。
→ **直接影响**: 我们通过代理访问 API，偶尔有网络抖动，cron 任务现在能自动重试。

### 8. Subagent completion 报告修复
- 子 agent 超时时现在显示 tool-only progress 而非空白
- blocked child-run 正确报错而非假报成功
- `sessions_spawn` child handoff 不再 false failure
→ **直接影响**: 我们的 team-lead workflow 依赖 subagent，之前偶尔丢失结果。

### 9. Dreaming 副作用修复
`dreaming.enabled=false` 时 recall tracking 不再写 dreaming 副产物。
→ **直接影响**: 如果我们临时关 dreaming，memory search 不会产生意外文件。

### 10. Heartbeat DM scope 路由修复
单 owner `session.dmScope=main` 的 exec/cron event wake 正确路由回 main session，不再 strand context 到 orphan DM queue。
→ **直接影响**: 我们的 heartbeat 每 30 分钟一次，context 不再意外分散。

## ⚡ 值得关注但暂不采用的

### 11. xAI/Grok web_search OAuth 复用
Grok 的 OAuth 可直接用于 web_search，如果我们接入 Grok 模型的话。

### 12. 子 agent bootstrap context 瘦身
默认只注入 `AGENTS.md` 和 `TOOLS.md`，不再带 persona/identity/memory。
→ **注意**: 可能影响 haru/ren 子 agent 的行为，升级后需验证。

### 13. Row-level session workflow helpers
Plugin SDK 新增行级 session 操作，deprecate `loadSessionStore`。
→ **未来**: 自定义 plugin 开发时有用。

### 14. OpenTelemetry smoke harness 扩展
支持 trace/metric/log export 验证，Prometheus/可观测性 smoke alias。
→ **未来**: 如果做生产监控可以用。

### 15. npm 包瘦身
排除文档图片/assets，安装包变小。升级自然享受。
