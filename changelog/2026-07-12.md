## Beta 2026.7.1-beta.5 新特性摘要

### 🆕 对我们有用的新功能

1. **Cron model selection** — Control UI 的 Quick Create 可以选择 agent-turn model，cron job 行列显示配置的模型。我们有 15+ cron jobs，这让管理和调试更直观。
2. **ClawRouter routing and quotas** — 内置 ClawRouter provider，credential-scoped 动态模型发现 + 用量/预算管理。可以统一管理多 provider 的 API key 和用量上限。
3. **Session organization and generated titles** — 自动为 session 生成标题 + unread state + rename/fork/archive 等管理功能。我们的 session 列表很长，有标题和分组会大幅改善可读性。
4. **External harness attachment** — `openclaw attach` 可以让 Claude Code 连接到已有 Gateway session，带 scoped MCP grants 和 TTL。对 ACP 工作流是巨大增强。
5. **Gateway host status** — Control UI Settings 显示主机信息（OS、内存、磁盘等），方便远程监控 kagura-server。

### 🔧 修复了我们遇到过的痛点

1. **Cron edit delivery** — 修复了编辑 cron job 的 delivery 设置时意外关闭 announce 的 bug。我们之前可能踩过这个坑。
2. **CJK Markdown emphasis** — 中日韩文字的 Markdown 强调标记不再泄露原始标记符号。我们大量使用中文输出。
3. **Discord streamed finals** — 完成回复作为新消息发送，让不活跃 channel 变为 unread。我们很多 channel 依赖 unread 标记。
4. **Agent empty replies** — 完成的 turn 没有可交付回复时显示可见失败。之前 plan-review 偶发空响应可能与此相关。
5. **Memory session repair** — `memory status --fix` 不再破坏 daily dreaming 状态。

### ⚡ 值得关注但暂不采用的

1. **Conversational onboarding (Crestodian)** — AI 引导的 setup 流程，对新部署有用，我们已配置好不需要。
2. **GPT-5.6 support** — Sol/Terra/Luna 模型家族支持，等正式发布后评估。
3. **Logbook work journal** — 配对节点截屏 → 私人时间线 + standup。有趣但 disabled-by-default，需要 paired node。
4. **Telegram Codex workflows** — Telegram 端 Codex 配对和 steering，我们目前不用 Telegram。
5. **Offline mobile chat** — iOS/Android 离线缓存 + Apple Watch 语音。Luna 的 iOS 端体验会改善但需要升级后才生效。

## 备注
Beta 5 changelog 极长（200+ changes），是一个大版本。建议等 stable release 后一次性升级，同时关注 ClawRouter 和 session titles 的采用时机。
