## v2026.5.7 — Cron/Channel/Plugin Polish

### 🆕 Useful New Features

1. **Cron `status` in JSON output** — `cron list --json` and `cron show --json` now include computed `status` (disabled/running/ok/error/skipped/idle). We can use this to simplify our toolchain-health cron status checks instead of reimplementing status derivation.
2. **`channels list` channel-only + `--all`** — Cleaner CLI output; model auth/usage details moved to dedicated commands (`models auth list`, `models list`). Better for scripted checks.
3. **Cached skill snapshot refresh on `/new`** — Skills list rebuilds after skill changes without gateway restart. Useful as we iterate on skills frequently.
4. **Discord cross-channel `message(action="send")` fix** — Provider-prefixed targets like `discord:channel:<id>` now work correctly. We've hit `Unknown Channel` errors on cross-channel sends from subagents — this fixes it directly.
5. **Compaction summary token clamping** — High-context compaction no longer requests invalid `max_tokens`. Prevents silent failures on long sessions.

### 🔧 Fixes for Pain Points

1. **Discord voice capture less choppy** — Post-speech silence grace extended to 2.5s with configurable `voice.captureSilenceGraceMs`. Not using voice yet but good to know it's improving.
2. **Cron delivery fail-fast** — Cron jobs with `delivery.channel=last` and no previous route now fail before spending tokens. Saves cost on misconfigured crons.
3. **Daily session rollover transcript persistence** — New transcript file persists when session ID changes on rollover. Fixes potential transcript loss we may not have noticed.
4. **Subagent `archiveAfterMinutes` config** — Completed subagents now honor `agents.defaults.subagents.archiveAfterMinutes` instead of hardcoded 5min. We spawn many subagents; configurable retention is useful.
5. **Owner enforcement for native commands** — Security hardening for native command handlers.
6. **Gateway task reconciliation** — Stale CLI run-context tasks can no longer block Discord reloads forever. May explain occasional channel reload delays.

### ⚡ Notable but Not Immediately Adopting

1. **`openai/chat-latest` model alias** — For trying ChatGPT Instant API. Interesting but we're stable on current models.
2. **WhatsApp LID forward mapping** — Proactive sends to LID-addressed contacts. We don't use WhatsApp proactive sends.
3. **Codex approval improvements** — `allow-always` decisions persist within session window. We don't use Codex.
4. **Discord voice channel permission audit** — Permissions checked before `/vc join`. Not using voice channels yet.
