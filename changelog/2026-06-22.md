## v2026.6.9 核心变化摘要 (422 merged PRs)

### 🆕 对我们有用的新功能（可立即采用）

| # | 变化 | 为什么对我们有用 |
|---|------|-----------------|
| 1 | **Codex 自动 plugin approvals** (#92625) | 我们用 Codex ACP 频繁，不再需要手动批准插件——减少 cron job 卡住 |
| 2 | **Standalone 官方 provider plugins** (#93470) | 外部 provider 独立 npm 包发布，Gateway 启动时自动发现——未来可按需装卸 provider |
| 3 | **Codex Hosted Search** (#93446) | 无需自配 API key 的搜索功能，cron 和 subagent 中可直接用 web_search |
| 4 | **Codex remote-node exec** (#93654) | 远程 node 的 exec 暴露为 Codex dynamic tool，macOS node 操作更方便 |
| 5 | **cron compact list** (#93395) | cron list 输出更紧凑——我们有 30+ job，查看更清晰 |
| 6 | **Codex app-server SecretRefs** (#94324) | 敏感信息可声明式传递给 Codex，不用写死在 prompt 里 |
| 7 | **Firecrawl keyless scrape** (#94551) | 无 key 也能用 firecrawl 抓取，研究型 cron job 受益 |
| 8 | **OTEL log export** (#94561) | 可观测性提升，运行时日志导出到 OpenTelemetry |

### 🔧 修复了我们遇到过的痛点

| # | 修复 | 关联痛点 |
|---|------|----------|
| 1 | **retry thinking-only errored turns** (#92191) | 我们遇到过 thinking-only 空回复卡住 session |
| 2 | **retry empty post-tool final turns** (#93073) | tool 调用后空回复被吞——见 6/18 日报 |
| 3 | **preserve fresh usage after compaction** (#93084) | compaction 后 usage 数据丢失——token 统计异常 |
| 4 | **drop partialJson artifacts from history** (#93469) | session history 偶现垃圾 JSON 片段 |
| 5 | **preserve pending subagent announces** (#94349) | subagent 完成通知偶尔丢失——gogetajob 流程断裂 |
| 6 | **cron default runMode "due" not "force"** (#94453) | 之前 cron 可能 force 执行导致意外重跑 |
| 7 | **cron isolated setup timeout retry** (#94588) | isolated cron setup 超时后会重试而不是直接 fail |
| 8 | **prevent lane timeout during long tool exec** (#94082) | 长时间 exec（如 git clone）不再被 lane 超时杀掉 |
| 9 | **Discord tool status emoji ordering** (#93488) | Discord 消息 emoji 覆盖问题修复 |
| 10 | **preserve CJK IME composition** (#93498) | WebChat 中文输入法组合不再被打断 |

## 升级建议

**推荐升级**：v2026.6.9 修复了多个我们实际遇到的痛点（thinking-only 空回复、post-tool 空回复、subagent 通知丢失、long exec 超时），新功能中 Codex auto approvals 和 cron isolated retry 对日常运行直接有益。

升级命令：
```bash
npm install -g openclaw@2026.6.9
openclaw gateway restart
```
