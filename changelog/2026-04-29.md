## 🆕 New Features We Can Adopt Now

### 1. Cron `maxConcurrentRuns` on dedicated lane (2026.4.26)
Parallel cron jobs no longer serialize on inner LLM execution — uses a dedicated `cron-nested` lane. **We run 15+ cron jobs; this directly unblocks concurrent execution bottlenecks we've seen.**

### 2. Cron isolated run timeout starts after lane entry (2026.4.26)
Queued cron/manual runs no longer spend timeout budget waiting in queue. **Fixes our occasional "cron timed out before doing anything" observations.**

### 3. `dreaming.model` config knob (2026.4.26)
Can now set a dedicated model for Dream Diary narrative subagents. **We run 12 dreaming sessions nightly — could use a cheaper model like Sonnet instead of Opus.**

### 4. Subagent task dedup — task in system prompt only (2026.4.26)
Delegated task now only appears in subagent system prompt (not duplicated as user message). **Saves tokens on every subagent spawn we do (dozens daily).**

### 5. Tool-loop detection ignores exec metadata volatility (2026.4.26)
Exec runtime metadata (PID, session, cwd) no longer resets loop detection. **Prevents false loop-detection resets in our heavy exec usage.**

### 6. `exec.timeoutSec` applies to background/yieldMs commands (2026.4.26)
Background and auto-backgrounded commands now respect configured timeout. **Prevents runaway background processes — we had this filed as a bug (#67600, credited to us!).**

### 7. Heartbeat/cron wakes as transient context (2026.4.25)
Heartbeat, cron, and exec wakeups no longer show as visible user prompts in transcripts. **Cleaner transcripts for our cron-heavy setup.**

### 8. Cerebras as bundled provider (2026.4.26)
New provider option. **Could evaluate for cost-sensitive cron jobs alongside DeepSeek.**

### 9. Asymmetric embedding config (2026.4.26)
`memorySearch.inputType` / `queryInputType` / `documentInputType` for embedding endpoints. **Useful if we switch to nomic-embed-text or other asymmetric models.**

### 10. Claude CLI importer (2026.4.26)
`openclaw migrate` can import Claude Code/Desktop configs. **Useful for onboarding new team members.**

## 🔧 Fixes for Pain Points We've Hit

### 1. Discord DM double-processing (#73441-adjacent fix in 2026.4.26)
`EPIPE` broken-pipe errors no longer crash Gateway after Discord delivery. **We saw occasional crashes after Discord sends.**

### 2. Cron failure classification fixed (2026.4.26)
Model/provider failures now properly increment error counters. Empty `NO_REPLY` tools treated as quiet success. Blocked commands show as errors not green. **Multiple cron reliability improvements we've needed.**

### 3. `sessions_spawn` model alias resolution (2026.4.26)
Bare model aliases now resolve using target agent's default provider. **Fixes potential model resolution issues in our subagent spawns.**

### 4. Stale bootstrap refresh on long sessions (2026.4.25)
`AGENTS.md`, `SOUL.md`, `MEMORY.md` changes now refresh on long-lived sessions. **We've had stale memory/agent configs after edits.**

### 5. Memory config hot-reload for `memory_search` (2026.4.26)
Provider changes via `config.patch` no longer leave stale embedding backends. **Fixes a real annoyance we've hit.**

### 6. Plugin startup with `PluginLookUpTable` (2026.4.26)
Major startup performance optimization — no more redundant manifest scans. **Faster gateway restarts for our 15+ cron setup.**

### 7. Compaction token estimate persistence (2026.4.26)
`/status` shows fresh context usage after auto-compaction. **We've seen stale token counts post-compaction.**

### 8. Gateway degraded mode for bad plugin config (2026.4.26)
Single broken plugin config no longer crash-loops Gateway. **Safety net for our setup.**

### 9. Reasoning tag leak prevention (2026.4.26)
Orphan closing reasoning tags can't leak chain-of-thought. **Privacy improvement for our public-facing channels.**
