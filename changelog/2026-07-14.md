## 🔥 2026.7.1 核心变化（对我们最有用的）

### 🆕 立即可采用的新功能

1. **Session 标题自动生成** — 新会话自动从首条消息生成简洁标题，支持 `utilityModel` 路由降成本。我们一直在等这个功能！多 channel + 多 cron 场景下 session 管理终于可读了。

2. **Cron 模型选择** — Control UI Quick Create 可直接选 agent-turn 模型，cron job 列表和详情页显示配置的模型。我们有 10+ cron jobs，之前看不出各 job 用什么模型。

3. **Exit-triggered schedules** — cron 新增退出触发调度，可在某 session 结束后触发下一个 job。对我们的 work-loop → plan-review 链式调度有用。

4. **Control UI sessions-first 导航** — session 成为主界面，带可搜索侧边栏、上下文环、reasoning-effort slider。管理我们 20+ 活跃 session 更方便。

5. **Cron edit delivery 修复** — 修复了 partial delivery update 时隐式 delivery mode 被覆盖的 bug。我们之前可能踩过这个：改 cron 的 bestEffort 时 announce 模式被关掉。

6. **CJK Markdown emphasis 修复** — 中日韩文字的加粗/斜体不再泄漏标记。我们大量使用中文 + Markdown，这个一直是小烦恼。

### 🔧 修复了我们遇到过的痛点

7. **Discord streamed finals** — 完成回复作为新消息发送，channel 变为未读。之前 Discord 不标未读让 Luna 容易漏消息。

8. **Cron 正确性大修** — 修复了超时时丢失 provider/model 选择、startup catch-up 丢失、blank thinking override 未清理等问题。我们 cron 密度高，这些修复直接受益。

9. **Delivery recovery pacing** — gateway 重启后的消息回放不再 burst 进 channel rate limit。之前重启偶尔导致 Discord 消息乱序。

10. **Agent empty replies 可见化** — 当一个 turn 没有可交付的回复时，显示明确的失败信息。之前偶尔"静默失败"，现在能看到。

### ⚡ 值得关注但暂不采用的

- **ClawRouter** — 内置路由插件，credential-scoped 模型发现 + 预算管理。我们目前用单一 provider，暂不需要。
- **Logbook work journal** — disabled-by-default 插件，把 node 截屏变成时间线日志。有趣但需要 paired node。
- **Conversational onboarding (Crestodian)** — AI 引导的首次设置。我们已配好，新用户受益。
- **iOS/Android 离线 + Apple Watch 语音** — 移动端大幅升级，但我们主要在 server 用。
- **GPT-5.6 成为默认** — 新模型默认，我们用 claude-opus-4-6 为主，可观望。
- **Claude Sonnet 5 / Mythos 5** — 新模型支持，可试用。
- **Telegram Codex 配对** — `/login` + `/steer` + `/tell`，我们不用 Telegram。
- **SQLite WAL 安全修复** — 要求有 patch 的 SQLite runtime，升级时注意 Node 版本兼容。
- **Gateway crash-loop recovery** — 重复启动失败后进入 control-plane-safe mode 而非无限重启。生产稳定性提升。
- **SecretRef 凭证隔离** — provider secrets 全程保持 sentinel 直到网络边界才注入明文。安全性提升。
