## v2026.5.6 (Hotfix)

Small targeted release — reverts a v2026.5.5 doctor repair that broke Codex OAuth routing.

### Key Changes

1. **Doctor/Codex OAuth revert** — The 2026.5.5 `doctor --fix` that rewrote `openai-codex/*` routes to `openai/*` is reverted. Could have broken OAuth-only GPT-5.5 setups. We don't use Codex but good to know.
2. **Plugin fetch symbol fix** — Third-party symbol metadata in request header dictionaries no longer breaks native `fetch` or `Headers`. Fixes plugin HTTP requests silently failing.
3. **Debug proxy header fix** — Same symbol metadata issue fixed for debug proxy replay.
4. **Web fetch timeout cleanup** — Timed-out fetches now return tool errors instead of leaving Gateway tool lanes stuck. We use `web_fetch` heavily in cron — this is relevant.

## v2026.5.5 (Major Patch)

Massive release with ~80+ fixes. Key items for our setup:

### 🆕 New Features We Can Adopt

1. **Discord live reasoning in progress drafts** — Reasoning text now shows in Discord progress drafts instead of bare `Reasoning` status. We use reasoning; this gives visibility.
2. **Gateway/process uptime in `/status`** — Compact uptime + host uptime visible from chat. Useful for our toolchain-health cron.
3. **Control UI sessions show agent runtime** — Sessions table shows runtime labels with filtering. Better visibility in WebChat.
4. **`/steer` through Discord authorization** — Plain text control commands like `/steer` now go through auth + mention gate properly instead of being silently dropped. We use `/steer` occasionally.

### 🔧 Fixes for Our Pain Points

1. **Discord heartbeat ACK timeout** — Heartbeat ACK measured from actual send, preventing false reconnect loops. We've seen Discord reconnect churn — this could be the fix.
2. **xAI/Grok reasoning effort** — Stops sending OpenAI-style reasoning controls to Grok models. We use `xai/grok-4.3` via OpenRouter — this fixes live run failures.
3. **Gateway shutdown timer cleanup** — Cancels delayed post-ready maintenance during close, prevents orphaned background timers. Explains some of our stale cron behavior after restarts.
4. **Hooks/session-memory off reply path** — `/new` and `/reset` no longer block channel reset replies on hook housekeeping. Reduces response latency.
5. **Plugin update for correction versions** — `2026.5.3-1` style versions now properly handled for plugin API ranges. We're ON this version — plugin installs may have been failing silently.
6. **TUI stale session hydration** — TUI no longer hydrates weeks-old transcripts before becoming responsive. We don't use TUI much but nice to know.
7. **OpenAI streaming first chunk** — SSE assistant role chunk sent immediately so cold agent setup doesn't leave clients with bodyless 200. Affects our OpenAI-compatible endpoint usage.
8. **Fireworks/Kimi thinking fix** — Manual model switches to Kimi no longer send rejected reasoning parameters. Relevant if we try Fireworks models.

### ⚡ Notable but Not Immediately Adopting

1. **Feishu topic thread fix** — First turns and follow-ups stay in same topic session. We don't use Feishu.
2. **Matrix approval retry** — Approval delivery retries with backoff. We don't use Matrix.
3. **iOS pairing improvements** — Private LAN and `.local` gateway support. We don't use iOS.
4. **Telegram Codex tool progress** — Deduplication of progress lines. We don't use Telegram.
5. **Docker container hardening** — Drops `NET_RAW`/`NET_ADMIN`, enables `no-new-privileges`. We run bare metal.
