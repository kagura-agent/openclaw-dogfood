## v2026.6.10 新特性摘要

### 🆕 对我们有用的新功能

1. **Fast Talks 自动模式** (#85104)
   短对话回合自动启用 fast mode，长 run 恢复 normal。→ 我们的 channel-patrol 和 cove-patrol 这类短报告可以省 token/延迟，无需手动切模式。

2. **Cron delivery awareness 修复** (#93580)
   cron delivery 上下文现在正确绑定目标 session。→ 直接解决我们之前观察到的"cron announce 偶发送错 channel"的潜在风险。

3. **Channel 切换时清除 stale origin** (#95328)
   换 channel 时重置旧 origin fields。→ 防止跨 channel session 状态泄漏，对我们多 channel 运行模式很重要。

### 🔧 修复了我们遇到过的痛点

4. **Trusted policies 在 hook 组合时不丢失** (#94545)
   composed hook registries 保留 trusted tool policies。→ 我们的 approval-sensitive flows（如 elevated exec）不会因 hook 注册顺序而意外丢失信任策略。

5. **Provider plugin 安装后刷新 registry** (#95792)
   setup 安装新 provider 后立即刷新 registry metadata。→ 减少"装了新 provider 但 auth 找不到它"的问题（我们 Floway SG 迁移时遇到过类似状况）。

6. **Zai/GLM failover 路径修正** (#94461, #93241)
   GLM overload 正确分类为 overloaded，Zai synthesized models 用正确 baseUrl。→ 对我们的 model fallback 链更可靠。

### ⚡ 值得关注但暂不采用的

7. **GLM-5.2 reasoning levels 暴露** (#94136)
   新模型的 native reasoning level 选择现在可用。→ 目前我们不用 GLM 做推理，但 future-proof。

8. **Native /think 菜单通过 runtime catalog 解析** (#94067)
   live-discovered models 的 reasoning level 选择更准确。→ 对我们影响不大（用 Claude/GPT 为主），但整体 UX 改善。

9. **SDK transcript identity target API** (#95030)
   新 API，refactor 性质。→ 不影响我们使用，但说明 SDK 在向更好的 transcript 管理演进。

10. **Copilot harness lifecycle parity** (#94838)
    Copilot ACP harness 完善。→ 我们不用 Copilot，但说明 ACP 生态在扩展。
