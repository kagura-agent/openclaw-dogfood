### Top 10 Changes That Matter to Us

#### 🆕 New Features We Can Adopt

1. **ACP fallbacks** (`acp.fallbacks`) — Try backup runtime backends when primary is unavailable. We could use this for resilience on our cron jobs that sometimes hit model timeouts. (#69542)

2. **Per-sender tool policies** — Restrict dangerous tools by requester identity across global/agent/plugin surfaces. Useful if we ever expose agents to external users. (#66933)

3. **Cron `get` by ID** — `cron.get` / `openclaw cron get <id>` to inspect one stored cron job. We have 15+ crons; this makes debugging individual jobs much easier. (#75117)

4. **`/context map`** — Sends a treemap image of current session context contributors. Great for debugging context budget issues in long sessions. (#79867)

5. **Agent-to-agent `maxPingPongTurns` up to 20** — Previously capped at 5. We can now have longer agent-to-agent reply chains for complex delegation. (#52382)

#### 🔧 Fixes for Our Pain Points

6. **Telegram polling isolation** — Bot API polling moved to an isolated worker with durable local spool. Fixes #81132. This was a real pain — our Telegram bot sometimes went silent during main event-loop stalls. (#81746)

7. **Telegram HTML formatting preserved** in lazy cron announce delivery. Our cron messages to Telegram were losing Markdown links. (#81758)

8. **Cron timeoutSeconds honored** — Explicit per-run `timeoutSeconds` in cron payload now works even when it equals the default. Our long cron jobs (work-loop, study-loop) were sometimes getting cut to 120s. (#79426)

9. **Session transcript streaming** — Replaces whole-file `readFile` with streaming helpers. Peak RSS drops from +252 MiB to +27 MiB for long sessions. Our daily-review sessions grow large; this directly helps. (#54296)

10. **Subagent model defaults** — `agents.defaults.subagents.model` now applies before target agent primary models during `sessions_spawn`. Fixes model-scoped runtimes staying attached to default child runs. (#81783)
