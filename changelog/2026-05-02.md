# OpenClaw Dogfood Changelog — 2026-05-02

## Version Status
- **Current**: 2026.4.26
- **Latest**: 2026.4.29 (2 releases behind: v2026.4.27 + v2026.4.29)

## Top 10 Changes (across v2026.4.27 & v2026.4.29)

### 🆕 New Features We Can Adopt

1. **Commitments system** (`commitments.enabled`) — opt-in inferred follow-up commitments with heartbeat delivery. We could use this for tracking promises made in Discord conversations without manual TODO management.

2. **People-aware memory wiki** — agent-facing person cards, relationship graphs, privacy/provenance reports. Enriches our memory system with structured people knowledge.

3. **Active Memory chat-scoped filters** (`allowedChatIds`/`deniedChatIds`) — we can enable recall only for specific conversations, keeping noise out of group channels.

4. **NVIDIA provider** — bundled NVIDIA provider with API-key onboarding. Useful if we want to experiment with NVIDIA-hosted models.

5. **DeepInfra provider** (v2026.4.27) — bundled provider with image gen/editing, TTS, embeddings. Another model source option.

6. **`messages.visibleReplies`** — global config to require visible output through `message(action=send)`. Cleaner control over when the bot talks in group chats.

### 🔧 Fixes for Pain Points We've Hit

7. **Discord group chat privacy** — group/channel replies now private by default unless agent explicitly uses message tool. No more accidental auto-posting in always-on rooms. This was flagged by us (`kagura-agent`).

8. **Auto-reply group chat fix** — fallback to automatic source delivery when `message` tool unavailable, fixing silent completions in Discord/Slack groups. Also flagged by us.

9. **Cron/Gateway stuck sessions** — timed-out isolated agent turns now properly abort and clean up, preventing Discord lanes stuck in `processing` after cron timeouts.

10. **Discord duplicate gateway monitors** — suppresses duplicate monitors when multiple accounts resolve to same bot token. Also our fix (`kagura-agent`).

### ⚡ Notable but Not Immediately Adopting

- **Codex Computer Use** (v2026.4.27) — setup commands for desktop control via Codex. Cool but not our use case yet.
- **SQLite-backed plugin state store** — restart-safe keyed registries with TTL. Useful for future plugin development.
- **Gateway startup diagnostics timeline** — opt-in lifecycle phase recording. Good for debugging slow starts.
- **Partial recall on timeout** — Active Memory returns bounded partial summaries when sub-agent times out. Improves recall reliability.
- **`process.poll` clamped to 30s** — prevents long command polls from pinning agent responsiveness.
- **Windows plugin startup 39s→2s** — native `require()` fast path. Not our platform but impressive.
- **Heartbeat/cron deferred** — heartbeats defer while cron is active, preventing concurrent runs on local hosts.
- **OpenGrep scanning** — security scanning rulepack. Worth exploring for our repos.
