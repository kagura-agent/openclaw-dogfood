## 🆕 对我们有用的新功能（可立即采用）

### 1. MCP tool result 自动类型转换
MCP 工具返回 `resource_link`、`resource`、`audio` 等非 text/image 类型时，自动转为 text，不再触发 Anthropic 400 错误。
- **为什么有用**：我们用多个 MCP 工具（Notion、Obsidian），之前偶尔遇到神秘 400，这下根治了。

### 2. Anthropic extended-thinking 会话恢复
prompt-cache 过期或 Gateway 重启后，extended-thinking 会话能自动恢复，不再卡死。
- **为什么有用**：我们重度使用 thinking mode，之前 Gateway 重启后偶尔要手动重开 session。

### 3. Parallel 作为内置 web_search provider
新增 Parallel 搜索引擎，`PARALLEL_API_KEY` 即可启用。
- **为什么有用**：多一个搜索后端选择，可以做 fallback 或对比。

### 4. config.patch 数组替换语义修复
`config.patch` 对数组的替换不再意外合并旧条目。
- **为什么有用**：我们用 config.patch 管理 tools.allow/deny，之前偶尔出现旧条目残留。

### 5. ClawHub GitHub 仓库技能安装
可以直接从 GitHub repo 安装 ClawHub skill，带 pinned commit 和策略检查。
- **为什么有用**：社区技能可以直接引用 repo，不必等官方 registry 发布。

## 🔧 修复了我们遇到过的痛点

### 6. QQBot/频道 thinking 标签泄漏
QQBot 发送前自动剥离 `<thinking>` 内容，不再泄漏内部推理到用户可见消息。
- **为什么有用**：虽然我们主要用 Discord，但 QQBot channel 也有用户，之前偶尔看到原始 thinking 输出。

### 7. Cron legacy JSON → SQLite 自动迁移
`doctor` preflight 现在会自动迁移旧 cron JSON 存储到 SQLite。
- **为什么有用**：升级后不再需要手动处理 cron 数据迁移。

### 8. macOS node 不再静默断开健康连接
macOS companion app 不再在已连接的情况下自行重连，减少 session churn。
- **为什么有用**：Mac 节点偶尔莫名断开的问题终于修了。

### 9. TUI 消息稳定性修复
optimistic user messages 不再在 history reload / abort 时消失或跳动。
- **为什么有用**：TUI 里发消息后偶尔看到「幽灵消息」，这个痛点终于解决。

## ⚡ 值得关注但暂不采用的

### 10. State/Storage 全面 SQLite 化
Matrix sync、memory-wiki、sandbox registry、ACPX process state 等全部迁移到 SQLite。
- **为什么关注**：说明 OpenClaw 在全面收敛存储后端，未来升级路径会更清晰，但也意味着升级要注意数据迁移。

### 11. Google Chat 原生审批卡片
Google Chat 审批改用平台原生卡片 UI。
- **为什么关注**：如果未来接入 Google Chat channel，审批体验会好很多。

### 12. 内联图片 data URL 脱敏
transcript 中的 data URL 和修复后的图片会被脱敏，防止原始 bytes 泄漏到存储/导出。
- **为什么关注**：安全加固，对我们的 session-logs 分析有好处。
