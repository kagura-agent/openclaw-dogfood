# OpenClaw v2026.5.27 Changelog (from v2026.5.26)

**Update available**: 2026.5.22 → 2026.5.27 (released 2026-05-28)

## 🆕 对我们有用的新功能

1. **OpenAI-compatible embedding provider 核心化** — 本地/hosted OpenAI 风格 embedding endpoint 现在是 core 支持，不再依赖插件。我们如果要用 self-hosted embedding（如 ollama/vllm），配置更简单了。(#85269)
2. **Pixverse 视频生成 provider** — 新增 Pixverse 视频生成 + API region 选择。如果我们需要 AI 视频生成，现在有原生支持了。
3. **ClawHub plugin 展示名** — catalog 和包列表现在用更干净的 display name，浏览插件体验更好。(#87354)
4. **Heartbeat runtime 模板独立** — heartbeat template 从 docs assets 拆出 + legacy 兼容修复，自定义 heartbeat 行为更清晰。(#85416)
5. **Plugin approval action metadata** — Plugin SDK 暴露审批 action 元数据，写自定义审批流程的插件更方便。(#87120)

## 🔧 修复了我们遇到过的痛点

1. **Cron prompt-cache affinity 稳定化** — isolated cron 的 prompt-cache 亲和性更稳定，我们 17+ 个 cron job 都能受益于更好的缓存命中率，减少 token 消耗。
2. **Session 读取性能优化** — read-only session metadata 使用借用而非拷贝，加上 plugin fingerprint 缓存、auto-enabled plugin config 缓存。hot path 减少重复发现，对我们高频 session 操作有帮助。(#86439)
3. **Codex runtime model 先解析再路由** — 避免 Codex model 走错通用路由，我们用 Codex ACP 时更稳定。(#87383)
4. **Telegram sendMessage durable delivery** — Telegram 消息发送现在走 durable outbound delivery，不再因瞬时网络问题丢消息。(#87261)
5. **Discord tool-warning artifacts 不再污染成功回复** — 之前偶尔有 tool warning 残留混入最终回复，现在清理了。
6. **Session write lock 超时释放** — 写锁超时后正确释放，避免 session 卡死。(#86123)
7. **Memory QMD search JSON 修复** — search 非零退出时也能 salvage 结果，memory search 更健壮。(#87225)

## ⚡ 值得关注但暂不采用的

1. **安全边界加强（群组 prompt / hostname / 命令注入）** — 多项安全加固：群聊 prompt 不再进 system prompt、hostname 点号规范化、侧效应命令包装器拦截、Tailscale 无认证暴露拒绝。我们是单用户场景影响不大，但安全升级值得肯定。(#87144, #87305)
2. **VLLM thinking params** — VLLM 现在支持 thinking 参数透传。等我们跑本地 VLLM 时有用。
3. **Claude CLI OAuth overlay for PI auth** — PI 认证 profile 的 Claude CLI OAuth 加载。我们暂不用 PI auth。
4. **OpenAI gpt-5.5 无缓存 catalog 解析** — 即使没有本地 catalog 缓存也能解析 gpt-5.5，模型路由更灵活。

## 升级建议

**建议尽快升级**，主要动力：
- Cron prompt-cache affinity 稳定化 = 降低 token 成本
- Session 读取性能优化 = 整体响应速度提升
- Codex runtime 修复 = ACP 更稳定
- 多项安全加固 = 安全底线提升
- Session write lock 修复 = 避免 session 卡死风险
