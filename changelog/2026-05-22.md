## v2026.5.20 新特性（2026-05-20 发布）

### 🆕 对我们立即有用的新功能

#### 1. Discord voice: 自动跟随用户进入语音频道
一句话：voice session 会自动跟随配置的用户进入语音频道，带 allowlist 检查和 multi-user handoff。
**为什么有用**：未来做语音交互时，不需要手动把 bot 拉进频道了。

#### 2. Discord voice: 自动注入 IDENTITY.md/USER.md/SOUL.md 到语音指令
一句话：realtime voice session 默认带上 identity 和人格上下文。
**为什么有用**：语音交互时 agent 有完整人设，不再"失忆"。

#### 3. Policy plugin: `openclaw policy` CLI
一句话：内置策略插件，支持 channel conformance 检查、doctor lint、workspace repair。
**为什么有用**：可以自动检查各频道配置是否合规，发现 drift。

#### 4. xAI device-code OAuth
一句话：xAI 支持 device-code 登录，headless 环境不再需要本地浏览器。
**为什么有用**：服务器上配置 Grok 模型更方便了。

#### 5. `localModelLean` per-agent 配置
一句话：可以只对单个 agent 开启 lean local-model 模式，不用全局。
**为什么有用**：精细控制资源分配，适合只有特定 agent 跑本地模型的场景。

### 🔧 修复了我们遇到过的痛点

#### 1. Cron: 成功的定时任务不再被诊断信息标记为失败
一句话：trailing tool warnings 不再让成功的 cron run 误判为 failed。
**为什么有用**：我们经常看到 cron 明明成功了却显示失败，应该就是这个。

#### 2. Cron: 后台 cron 不再阻塞人工 chat
一句话：cron 跑在独立 wake lane 上，不再 block 主 session。
**为什么有用**：之前偶尔等 cron 跑完才能聊天，这是 root cause。

#### 3. 心跳消息不再污染 context
一句话：heartbeat no-op 消息从 embedded context snapshots 中过滤掉了。
**为什么有用**：长 session 里偶尔看到奇怪的心跳残留，会干扰模型理解。

#### 4. Subagent: stale completion 恢复
一句话：stale completion announces 会重试，requester run stale 时走 message-tool handoff。
**为什么有用**：subagent 完成但 parent 没收到结果的问题。

#### 5. Approvals: `/approve` 路由修复
一句话：手动 `/approve` 现在正确路由到 approval runtime，不再显示 unknown/expired。
**为什么有用**：之前偶尔 approve 不生效要多试几次。

#### 6. Status: 显示 session pinned model 差异
一句话：`/status` 现在会显示当 session model 与默认不同时的提示和文档链接。
**为什么有用**：排查"为什么这个 session 用了不同模型"更容易了。

### ⚡ 值得关注但暂不采用

#### 1. OpenRouter provider-level routing policy
关注原因：如果用 OpenRouter 做 fallback，可以设置 provider 级别路由偏好。

#### 2. Codex harness 升级到 `@openai/codex` 0.132.0
关注原因：ACP Codex agent 能力更新，自动享受。

#### 3. Exec skill allowlist 收紧
注意：skill 文件必须用 read tool 加载，旧的 `cat SKILL.md && ...` 兼容路径已移除。升级后需确认 skill 调用正常。
