# OpenClaw Changelog — 2026-05-19

## 版本变化
- **当前版本**: 2026.5.18 (50a2481)
- **最新版本**: 2026.5.18
- **状态**: ✅ 已升级！从 2026.5.3-1 跳到 2026.5.18，跨越 15 个 daily releases

## 🆕 对我们有用的新功能（可立即采用）

1. **defineToolPlugin + plugins build/validate/init** — 类型化的简单 tool plugin SDK，比之前手写 manifest 方便很多。我们自定义工具可以迁移过来。

2. **Browser dialog handling** — `browser dialog --dialog-id` 可以处理弹窗，之前 browser automation 遇到 alert/confirm 就卡住，现在有正式解法。

3. **Meme-maker skill** — 内置 meme 模板搜索 + 本地 SVG/PNG 渲染 + Imgflip hosted rendering。可以替代我们之前的 agent-memes workaround。

4. **Presentation rich messages** — 支持 buttons/selects/context blocks 的结构化消息。Discord channel 里的交互式内容终于有 first-class 支持。

5. **Gateway restart overlap optimization** — startup logging 和 plugin-service startup 并行，restart ready latency 降低。我们频繁 restart 时感知会更好。

6. **Memory incremental sync** — 启动时只扫描新/变更的 transcript 文件做增量索引，不再全量重建。memory 启动速度应该有明显提升。

7. **`openclaw sessions list` alias** — 之前只有 `openclaw sessions`，现在 `openclaw sessions list` 也可以了，CLI 一致性更好。

## 🔧 修复了我们遇到过的痛点

1. **Subagent completion delivery** — 多个修复：parent runs 不再在 child wait timeout 后丢失结果、keep-mode 结果不再在 retry 后丢失、group/channel subagent completions 正确路由。我们之前遇到的 subagent 结果丢失问题应该好很多了。

2. **Cron/Discord announce mode** — 修复了 cron 定时发送时既走 message-tool 又走 automatic visible reply 导致重复发送的问题。

3. **Telegram forum topic preservation** — topic ID 在 handoff 中不再丢失。我们的 Telegram forum 回复之前有时会跑到错误的 topic。

4. **Discord final reply delivery in progress-mode** — preview stream 结束后 final reply 不再被 deduplicate 掉。之前偶尔看到的"回复消失"应该是这个 bug。

5. **Message tool stale context routing** — 当 message tool 有 stale webchat context 时优先用非 webchat 路由。修复了我们遇到的消息投递到错误 channel 的问题。

6. **Subagent sandbox-peer routing** — 修复 completion announcement 路由回正确的 originating session。

7. **Memory QMD hyphenated query** — 带连字符的日期/标识符搜索不再错误 fallback 到 builtin index。

## ⚡ 值得关注但暂不采用

1. **Android Talk Mode realtime voice** — Android 端切换到 realtime Gateway relay voice sessions。我们暂时没有 Android 使用场景。

2. **Codex app-server 大量修复** — GPT-5.1/5.2/5.3 支持、native code mode 修复等。我们目前用 Claude，但如果将来切 OpenAI 这些很重要。

3. **QA-Lab runtime parity** — 新的 Codex-vs-Pi runtime parity 测试框架。对我们自己的质量保证可以参考方法论。

4. **xAI OAuth PKCE** — xAI provider 的 OAuth 修复。我们没用 xAI 但说明 provider 生态在扩展。

5. **HTTPS managed forward-proxy** — `proxy.tls.caFile` 支持。如果我们的 proxy 需要 TLS 可以用。
