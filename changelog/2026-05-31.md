## v2026.5.28 新特性摘要

### 🆕 对我们有用的新功能（能立即采用）

1. **Cron 速率限制重试** — cron job 遇到 model rate limit 时会立即重试，而不是等下一个调度窗口。我们有 9 个 cron job，rate limit 碰撞是真实问题，这个直接减少 skipped run。
2. **Workboard 协调工具** — 新增 agent coordination tools，用于跟踪和交接活跃 agent 工作。对我们多 cron + subagent 的工作流很有用，可以避免重复工作。
3. **Status 显示活跃 subagent 详情** — `/status` 现在展示 spawned subagent 的详细信息，调试 work-loop 和 abti-loop 的 spawn 链更方便。
4. **Discord progress draft 显示评论** — live Discord run 会暴露有用的 in-progress context，我们的 Discord channel 交互体验会更好。
5. **Claude Opus 4.8 支持** — 新模型可用，可以评估是否切换。
6. **加密 PDF 提取** — ClawPDF 现在支持加密 PDF，study-loop 遇到加密论文时不再卡住。

### 🔧 修复了我们遇到过的痛点

1. **Session lock 超时释放** — 之前 cron session 偶尔卡死可能就是这个原因（release session locks on timeout abort），runtime teardown 不再删除 live lock。
2. **Subagent cwd/workspace 分离** — spawned agent 保持独立的工作目录状态，避免 work-loop spawn 的 Claude Code session 互相干扰。
3. **Edit 参数智能引号修复** — model 生成的 tool call 带 smart quote 时能自动修复，不再 corrupt 有效输入。我们日常 edit 调用量很大。
4. **Stale restart continuation 避免** — 重启后不再复用过期的 continuation，减少诡异行为。
5. **Codex app-server/helper 失败不拖垮共享 runtime** — work-loop 的 Codex 稳定性提升。

### ⚡ 值得关注但暂不采用

1. **GitHub Copilot agent runtime + Codex Supervisor plugin** — 新的 ACP runtime 选项和委托 Codex workflow，等稳定后可以评估替换部分 work-loop 流程。
2. **Policy 比较和合规检查** — ingress-channel conformance、sandbox-posture conformance，安全审计可以用但不急。
3. **iOS Pro UI 刷新** — 移动端体验大幅改善（Command, Chat, Agents, Settings, hosted push relay, Talk playback），但我们主要用 server 端。
4. **Plugin SDK reply payload hook** — plugin 开发者向，暂时不需要。
5. **性能优化一揽子** — install-record cache、native JSON parsing、tool-search catalog 复用、session patch writer 等，升级后自动生效。
