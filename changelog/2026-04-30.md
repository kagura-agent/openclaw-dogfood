## New in 2026.4.27

### 🆕 New Features We Can Adopt Now

#### 1. DeepInfra bundled provider
Full provider with model discovery, image gen/editing, TTS, embeddings, video. **We could use this as a fallback provider or for cheap embedding workloads alongside Ollama.**

#### 2. Operator-managed outbound proxy (`proxy.enabled` + `proxy.proxyUrl`)
First-class proxy config instead of env vars. Strict http:// validation, loopback bypass, cleanup on exit. **We currently use `https_proxy` env hacks in cron jobs — this is the proper replacement.**

#### 3. Codex Computer Use setup (`/codex computer-use status/install`)
Desktop control through Codex with marketplace discovery and fail-closed MCP checks. **Not immediate priority but opens browser automation through Codex path.**

#### 4. `models.pricing.enabled` toggle
Offline/restricted installs can skip startup pricing catalog fetches. **Could reduce our Gateway startup time if we disable it (we don't use OpenRouter pricing).**

#### 5. Cron `delivery.threadId` in schema
Scheduled announcements can now target Telegram forum topics and threaded destinations properly. **We use Telegram topics — this is a proper fix for threaded cron delivery.**

#### 6. Plugin manifest-first model catalogs
Provider rows/aliases/suppressions now declared in manifests, reducing Gateway boot work. **Should make our startup faster automatically after upgrade.**

### 🔧 Fixes for Pain Points We've Hit

#### 1. Memory compaction flush no longer leaks into transcripts
Pre-compaction memory-flush prompts stay runtime-only, no longer visible in `chat.history`. **We've seen these mystery "user" turns in session exports — fixed!**

#### 2. `sessions_yield` subagent state preserved properly
Parent sessions wait for final post-compaction answer instead of getting `(no output)`. **We spawn dozens of subagents daily — this fixes intermittent empty results.**

#### 3. Cron agentId inference fixed
`cron.add` now infers the creating session's agentId when omitted. **Our cron jobs should route more reliably.**

#### 4. Discord group chat replies private by default
Always-on rooms won't leak automatic output; `visibleReplies: "automatic"` to restore old behavior. **Not directly relevant (we control our channels) but good default.**

#### 5. Gateway startup — channels start without waiting for model prewarm
Slack/Telegram/Discord come online promptly even when provider discovery is slow. **We've seen slow Discord reconnects after restart — this should help.**

#### 6. Dream Diary timeout increased to 60s
Slower models can finish narrative generation (was 15s). **We run 12 dreaming sessions nightly — this prevents timeout-based failures.**

#### 7. Dream Diary fan-out capped
Multi-workspace dreaming no longer spawns unbounded subagents. **Directly relevant to our heavy dreaming usage.**

#### 8. Slack socket mode pong timeout configurable
Fixes stale websocket detection. **We don't use Slack but good to know.**

#### 9. Auto-reply drain timeout
Never-settling tool-result tasks no longer leave sessions active forever. **We may have had ghost sessions from this.**
