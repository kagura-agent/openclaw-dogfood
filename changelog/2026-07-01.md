## v2026.6.11 重要变化

### 🆕 对我们有用的新功能

1. **Slack relay mode** (#94707) — Slack incoming message 可用 relay 模式转发。我们虽然主用 Discord，但如果未来接 Slack workspace 这是基础能力。

2. **`openclaw agent --message-file`** (#93351) — 可以从文件读取 message 发给 agent。对批量测试、脚本化注入消息非常实用，比如可以把复杂 prompt 存文件后直接喂给 agent。

3. **per-DM model override** (#95120) — 可按 DM 用户设置不同 model。如果有人在 DM 里需要不同模型能力（如用便宜模型回答简单问题），这个很有用。

4. **Mattermost `/oc_queue` native slash command** (#95546) — Mattermost 原生命令支持，我们不用但对社区有价值。

5. **per-agent usage-cost reporting** (#94483) — 可以按 agent 查看 usage cost。对我们多 cron job 场景下追踪哪个 job 烧钱最多非常有用。

6. **RAFT CLI wake bridge** (#95497) — 新的远程唤醒通道。可以从外部系统触发 agent wake-up。

### 🔧 修复了我们遇到过的痛点

7. **Cron no-config delivery validation** (#95754) + **thread-aware dedupe** (#95794) + **pending recurring runs preserved on add/remove** (#94323) — cron 三连修。我们大量依赖 cron，之前偶尔遇到 delivery 配置不对的 job 默默失败，以及 add/remove 操作偶尔丢掉正在 pending 的 run。这些都修了。

8. **Stuck release_lane / Telegram inbound block** (#95299) — 释放锁被活跃 worker 持有时变成 no-op 的 bug。虽然我们不用 Telegram，但类似的 lane-blocking 问题理论上也影响其他 channel。

9. **Aborted runs stop cleanly** (#94412) — 之前 abort 后 tool run 可能继续循环。我们偶尔 abort subagent 时遇到过残留执行。

10. **Long-context tool-result prompts cache-stable** (#95624) — 长 context 下 tool result 的 prompt cache 不稳定问题修复。我们的 cron session 经常跑到高 token count，这个直接影响效率。

### ⚡ 值得关注但暂不采用的

11. **Android settings detail panels** (#95148) — 移动端配置体验提升，我们不用移动端管理但说明移动端在成熟。

12. **Codex partial deltas as partials** (#95404) — Codex ACP harness 的流式改进。我们用 ACP 但主要走 Claude Code，不过 Codex harness 改进说明 ACP 生态在完善。

13. **Externalized official plugins + icon manifest** (#95683, #95845) — 插件分发更干净了。等我们有自定义 plugin 需求时会受益。

14. **Memory artifact sanitization** (#95791) — memory 保存时做清洗。减少垃圾进入 memory store 的风险。

15. **Plugin npm update breaks running gateway** (#95589) — 修复了 npm 更新插件时正在跑的 gateway 的 import 会坏掉。我们升级时应该更安全了。
