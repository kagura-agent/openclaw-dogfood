## v2026.7.1-beta.1 新特性摘要

### 🆕 对我们有用的新功能

1. **`on-exit` cron schedule** (#92037) — 新的 cron 调度类型，当被监控的命令退出时唤醒 agent。适合我们的长时间任务场景（如 Claude Code 子任务完成后自动触发后续步骤），替代 poll 循环。
2. **`openclaw attach`** (#96454) — 将外部 harness 挂载到已存在的 Gateway session，可以交互式恢复/检查 Codex 工作流。调试 subagent session 时极有用。
3. **Scoped conversation capability profiles** (#98536) — 为单个对话设置 tool/access 边界，不影响全局 default profile。可以给特定 channel 限制工具权限（如 #finance 禁用 exec）。
4. **Cursor Agent 作为 autoreview engine** (#97348) — 新增 Cursor Agent 作为代码审查引擎选项。我们的 code-review skill 可以考虑集成。
5. **Built-in usage footer** (#92657, #92877) — 每轮对话自动附加 token 用量信息，无需手动 /usage。在 Telegram 上也做了换行优化。

### 🔧 修复了我们遇到过的痛点

6. **Cron provider/model 超时保留** (#95943) — 之前 isolated run 超时后 provider/model 设置会丢失，现在保留。我们的 cron job 遇到过类似问题。
7. **隔离 cron sessions_send 反馈回路修复** (#92283) — 隔离 cron 通过 sessions_send 发消息后不再注入回自身 session，避免上下文污染。直接影响我们的 A2A 通信模式。
8. **Gateway run-cache 增长上限** (#77973) — 防止高并发 run fan-out 导致内存无限增长。我们 12+ cron job 并发场景下很重要。
9. **Codex get_goal/sessions_spawn 误报失败修复** (#98659, #96856) — 之前这些工具调用被错误记录为 failure。我们大量使用 goal 和 subagent spawn。
10. **Subagent 重启时保留 steered task 文本** (#77539) — subagent restart 后任务描述不再丢失。我们的 work-loop 重度依赖 subagent。
