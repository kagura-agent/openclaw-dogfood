# OpenClaw Changelog - 2026-05-06

## Version Update: 2026.5.3-1 → 2026.5.4

Massive patch release (~200+ items). Highlights for our setup:

### 🆕 New Features We Can Adopt Now

1. **`models auth list` CLI** — `openclaw models auth list [--provider <id>] [--json]` lets us inspect saved auth profiles without dumping secrets. Useful for debugging multi-provider setups.

2. **Plugin skills directory** — Plugin-declared skills now publish to `~/.openclaw/plugin-skills/`, making them discoverable by agent file-based skill scanning. Our custom skills remain in workspace, but plugin skills (browser, etc.) are now auto-linked.

3. **`streaming.progress.render: "rich"` for Slack** — Block Kit rich progress drafts backed by structured progress data. If we ever use Slack, this is ready.

4. **`toolProgressDetail: "raw"` agent override** — `agents.defaults.toolProgressDetail: "raw"` shows raw command output in progress drafts for debugging. Great for dev/debug mode.

5. **`streaming.preview.commandText: "status"`** — Hides command/exec text in preview progress lines while keeping status. Reduces noise in Discord tool progress.

6. **`sessions` CLI pagination** — `openclaw sessions --limit <n|all>` with pagination metadata. Prevents unbounded output on our large session store.

7. **`registerIfAbsent` for plugin runtime state** — Atomic keyed-store dedupe claims for plugins. Useful if we build custom plugins.

### 🔧 Fixes for Pain Points We've Hit

1. **CalVer correction version handling** — `2026.5.3-1` now properly treated as stable and newer than base release. We were ON this version; explains why plugin updates may have been awkward.

2. **Cron stall diagnostics** — Failed cron runs now report actual tool-policy failures instead of misleading green results. We've seen silent cron failures before.

3. **Discord IPv4 preference** — Discord REST and gateway now prefer IPv4 for startup. Fixes stalls on IPv4-only networks (we're on a VPS).

4. **Discord failed reply detection** — Failed final reply delivery now correctly shows as failed instead of "done". We've seen ghost replies.

5. **Memory `corpus=all` search balance** — Memory hits no longer starved by higher wiki scores. Our memory searches should be more balanced.

6. **Active Memory bounded search query** — Channel/runtime metadata no longer becomes the memory search string. This was a known bug.

7. **Subagent completion announce fix** — Prefix-only completion announce replies now fall back to captured child result. We've seen truncated subagent reports.

8. **Prompt cache restoration** — Per-turn runtime context no longer breaks prompt cache on chat continuations. Cost savings.

### ⚡ Notable but Not Immediately Adopting

1. **Google Meet agent-mode voice** — Major overhaul of Meet voice bridge with Twilio, realtime transcription, echo suppression. We don't use Meet.

2. **WhatsApp Channel/Newsletter targets** — `@newsletter` outbound. We don't use WhatsApp channels.

3. **Exec approval shell explainer** — Tree-sitter-backed command explainer for approval surfaces. Improves security UX.

4. **Plugin `before_agent_finalize` retry** — Workflow plugins can request one more model pass. Interesting for FlowForge evolution.

5. **DeepSeek V4 `xhigh`/`max` thinking** — OpenRouter mapping fixed. We use OpenRouter for some models.

6. **Sandbox per-shard registry** — Reduces session lock contention. Performance improvement for heavy subagent use.
