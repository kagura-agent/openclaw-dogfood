# Adoption Recommendations — 2026-06-21

## 无新版本，关注现有功能深化

### 1. 优化 version-check cron timeout 策略
- **现状**: 昨日因 LLM 超时导致整个 run 失败（534s）
- **建议**: 将 `timeoutSeconds` 从 900 调至 600（10min），因为本 job 不需要复杂操作。或在 payload 中加 `lightContext: true` + 精简 prompt
- **收益**: 避免卡住占用资源
- **状态**: lightContext 已启用，可观察是否还超时

### 2. 考虑 cron job failureAlert 配置
- **现状**: version-check 超时后 lastFailureNotificationDeliveryStatus = "not-delivered"
- **建议**: 为关键 cron 配置 `failureAlert` 属性，确保失败时有通知
- **收益**: 及时发现 cron 异常
- **优先级**: 低 — 多数 job 运行稳定

### 3. exec env 安全策略适配
- **现状**: v2026.6.8 禁止通过 exec env 传 HTTPS_PROXY
- **建议**: 移除 cron prompt 中关于 `export https_proxy=...` 的指示，改用系统级 proxy 配置
- **收益**: 避免无效的 security violation 错误
- **优先级**: 中 — 影响需要 proxy 的操作
