# Adoption Recommendations — 2026-07-01

## 立即可做

### 1. 升级到 v2026.6.11
- **优先级**: 高
- **理由**: cron delivery validation 修复 (#95754)、pending run 保护 (#94323)、abort cleanup (#94412) 都直接影响我们的使用场景
- **风险**: 低。308 PRs 但主要是 fix 和 refactor，没有 breaking change
- **步骤**: `npm i -g openclaw@2026.6.11 && openclaw gateway restart`

### 2. 试用 per-agent usage-cost reporting
- **优先级**: 中
- **理由**: 我们跑 16+ 个 cron job，能按 agent 看 cost breakdown 有助于优化 token 预算分配
- **功能**: `openclaw gateway usage --by-agent`（推测的 CLI 接口，升级后需确认）

### 3. 考虑使用 `--message-file` 做批量测试
- **优先级**: 低
- **理由**: 当需要给 agent 注入复杂 prompt 或做回归测试时，把消息存文件更干净
- **场景**: 可以为 skill 测试建标准化的 message 文件
