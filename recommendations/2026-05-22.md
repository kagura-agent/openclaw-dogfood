# 采用建议 — 2026-05-22

## 🔴 高优先级

### 1. 升级到 2026.5.20
落后 2 个版本了。v2026.5.19 和 v2026.5.20 都有我们直接受益的修复：
- cron 成功任务误标失败 → 直接影响我们的日常 cron 监控
- cron 不再阻塞人工 chat → 改善交互响应
- heartbeat 不再污染 context → 长 session 质量提升
- `/approve` 路由修复 → exec 审批更可靠
- subagent completion 恢复 → 减少 parent 丢结果

**建议**: 本周末安排升级，先在非高峰时段执行 `npm i -g openclaw@latest`

### 2. 排查 auto-trader NO_REPLY
连续出现空回复，需要查 session transcript 确认是 tool error、timeout 还是模型拒绝。

## 🟡 中优先级

### 3. 尝试 Policy plugin
v2026.5.20 新增的 `openclaw policy` 可以检查频道配置合规性。
**场景**: 用在 daily-audit 里，自动扫描各 channel 配置 drift。

### 4. voice session identity 注入
如果后续做语音交互实验，v2026.5.20 自动注入 IDENTITY/SOUL 到 voice instructions。
**场景**: Discord voice 频道的 agent 交互。

## 🟢 低优先级

### 5. `defineToolPlugin` SDK
v2026.5.19 的 plugin SDK 成熟度提升。
**场景**: finance tool plugin 如果要从 MCP 迁移到原生 plugin，现在有了正式路径。

### 6. Config reload metadata
v2026.5.20 的 config lookup 能告诉你改配置是否需要重启。
**场景**: 升级后自然享受，不需要额外配置。
