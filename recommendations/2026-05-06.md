# Adoption Recommendations - 2026-05-06

## Immediate (upgrade to 2026.5.4)

### 1. Upgrade to 2026.5.4
- **Why:** Fixes CalVer correction version handling (we're on 2026.5.3-1), Discord IPv4 preference, memory corpus balance, subagent announce truncation, prompt cache
- **Risk:** Low — patch release, mostly fixes
- **Action:** `openclaw update`

### 2. Enable `streaming.preview.commandText: "status"`
- **Why:** Our Discord progress drafts show raw exec commands which is noisy. This new option shows only status text.
- **Where:** `agents.defaults` or per-agent config
- **Impact:** Cleaner Discord tool progress messages

### 3. Use `openclaw models auth list`
- **Why:** We have multiple providers (OpenRouter, Anthropic, etc.). This lets us audit auth profiles without touching secrets.
- **Action:** Add to toolchain-health cron for auth profile monitoring

## Short-term (this week)

### 4. Explore `toolProgressDetail: "raw"` for debugging
- **Why:** When debugging cron failures or subagent issues, raw tool output in progress drafts would help
- **Where:** Per-agent override for debug sessions, not default
- **Config:** `agents.<agent>.toolProgressDetail: "raw"`

### 5. Plugin skills directory
- **Why:** Plugin-owned skills (browser, etc.) are now auto-published to `~/.openclaw/plugin-skills/`. Verify our skill discovery works with both workspace and plugin skill paths.
- **Action:** Check `ls ~/.openclaw/plugin-skills/` after upgrade

## Future consideration

### 6. `registerIfAbsent` for plugin state
- **Why:** If we build custom plugins (e.g., FlowForge as a plugin), atomic state claims would be useful for coordination
- **When:** When plugin development becomes a priority
