# 采用建议 — 2026-06-13

## 基于 v2026.6.6 的采用建议

### 1. 🔴 升级到 2026.6.6
- **理由**: Cron 修复直接影响我们 — 任务取消状态保留、delivery 失败恢复。我们有 12+ 个活跃 cron，这些修复降低了静默失败风险。
- **风险**: 低 — 主要是修复和安全加固，无破坏性变更
- **建议**: 本周内升级，先在非高峰时段测试

### 2. 🟡 启用 diagnostics channel tool I/O 捕获
- **功能**: trusted diagnostics channels 现在可以看到完整的 tool 输入输出
- **场景**: 调试 code-review subagent 失败时，能看到每个 tool call 的完整内容，而不是猜测
- **优先级**: 中 — 等升级后配置

### 3. 🟡 给关键 cron 配置 failureAlert
- **延续昨天建议**: 升级后 cron 的 failure 状态更可靠了，正好配合 failureAlert 使用
- **目标 cron**: version-check, 以及其他关键定时任务
- **配置**: failureAlert.after=2, 投递到主 channel

### 4. 🟢 评估 ClawHub dry-run 改进
- **功能**: dry-run 跳过 publish approval
- **场景**: 我们有多个自定义 skill，发布流程会更顺畅
- **优先级**: 低 — 等有 skill 发布需求时自然受益
