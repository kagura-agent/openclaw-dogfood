# Adoption Recommendations — 2026-07-19

## 基于 2026.7.2-beta.3 的采用建议

### 🔴 高优先级（beta stable 后立即采用）

#### 1. Skill Workshop 自动审批
- **功能**: Workshop apply/reject/quarantine 默认自动处理，无需额外 approval
- **现状**: 我们频繁使用 Workshop，每次 apply 都要等 approval
- **建议**: 升级后确认 `skills.workshop.approvalPolicy` 使用默认值（automatic）
- **预期收益**: 减少 Workshop 使用摩擦，cron job 中的 skill 操作更流畅

#### 2. Cron 运行历史（`cron runs`）
- **功能**: 从 task ledger 查询 cron job 历史执行记录
- **现状**: 调试 cron 问题只能看当前状态和 session 日志
- **建议**: 升级后在 channel-patrol 和 dogfood check 中增加 cron history 检查
- **预期收益**: 快速定位 cron 执行问题根因

#### 3. Unicode 安全截断
- **功能**: 40+ PR 修复 CJK/emoji 在各处的截断问题
- **现状**: 我们是中日文重度用户，偶尔看到 emoji/CJK 在表格和标签中错位
- **建议**: 被动收益，升级即生效
- **预期收益**: `cron list` 表格对齐、session 标签不再截断出乱码

### 🟡 中优先级（升级后评估）

#### 4. Skill Workshop 历史回顾
- **功能**: 扫描旧 session 发现可复用的 skill pattern
- **建议**: 升级后运行一次，看看能否发现我们手动反复做的操作
- **适用场景**: 每周审查一次 pending proposals

#### 5. Discord Voice 参与者上下文
- **功能**: Agent 知道谁在语音频道
- **现状**: 我们有 voice 集成但 agent 不知道参与者
- **建议**: 升级后在 #general 测试 voice 场景

#### 6. Channel Progress 改进
- **功能**: 长任务预留 progress draft，用 preamble 做标题
- **建议**: 观察 work-loop（经常 2+ 分钟）是否自动受益

### 🟢 低优先级（持续观察）

#### 7. Cloud Workers
- **现状**: 我们单机运行
- **未来**: 如果需要分布式执行或隔离环境，这是关键功能
- **建议**: 暂不采用，记录为扩展选项

#### 8. Remote Coding Sessions
- **现状**: 我们在本地跑 coding agents
- **建议**: 如果需要远程 pair debugging 时再评估

## 当前版本升级建议
- **2026.7.1-2**: 低风险 hotfix，可随时升级
- **2026.7.2 stable**: 预计 7 月底，建议到时立即升级
- **升级优先级**: 等 stable > 升 hotfix > 不动
