# 采用建议 — 2026-05-16

## 高优先级（升级后立即采用）

### 1. 使用 `/context map` 调试 token 问题
**场景**: 我们的长 session（work-loop, study-loop）经常接近 context 上限
**操作**: 在 cron 超长运行后手动执行 `/context map` 分析 token 分布
**收益**: 可视化发现哪些 tool result 或 memory 占了太多 context

### 2. 配置 `acp.fallbacks`
**场景**: Claude 偶尔 503/429 时 cron 任务失败
**操作**: 在 config 中添加 `acp.fallbacks` 指向备用 provider
**收益**: cron 任务不因单 provider 故障中断

### 3. 给 subagent 配置显式 timeout
**场景**: work-loop subagent 有时运行 10min 超时（见今日 abti-loop timeout）
**操作**: 在 spawn 时传入合理的 `timeoutSeconds`，cron payload 也显式设置
**收益**: 利用 #79426 修复后的正确 timeout 行为

## 中优先级

### 4. 审查 per-sender 工具策略
**场景**: Discord 群里社区成员可能触发危险操作
**操作**: 研究 `tools.byProvider` 和 per-sender policy 配置
**收益**: 更精细的安全边界

### 5. 启用 Control UI auto-scroll "always follow" 模式
**场景**: 监控长运行 session 时经常丢失输出
**操作**: 在 Control UI 中切换为 "always follow"
**收益**: 流式输出不再需要手动滚动
