# Adoption Recommendations - 2026-05-13

**Priority: HIGH**. We're 3 patch versions behind. Key benefits:
- Cron JSON status output — could improve our toolchain-health checks
- Skill cache fix — prevents stale skill lists in long-lived cron sessions

**Action**: Schedule upgrade during low-activity window.

## 2. Use `cron list --json` Status Field (post-upgrade)

v2026.5.7 adds computed `status` to cron JSON output. Our toolchain-health cron currently checks cron health manually. Post-upgrade, we could simplify:
```bash
openclaw cron list --json | jq '.[] | select(.status == "error")'
```

## 3. Consider `openai/chat-latest` for Model Testing

v2026.5.7 adds support for `openai/chat-latest` alias. Our abti-loop does model benchmarking — this could be useful for tracking ChatGPT API changes without hardcoding versions.

## 4. Subagent `archiveAfterMinutes` Tuning (post-upgrade)

v2026.5.7 respects `agents.defaults.subagents.archiveAfterMinutes` config instead of hardcoded 5min. Our workloop subagents sometimes run 2-3 minutes but abti subagents can run much longer. Consider setting this to 30min to keep registry cleaner while accommodating long runs.

## 5. Monitor Beta for `/context map` Treemap

The v2026.5.10 beta includes visual context treemap. Would be valuable for understanding why some cron sessions (github-check at 86K tokens) use more context than expected. Adopt when stable.
