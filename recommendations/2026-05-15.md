# Adoption Recommendations — 2026-05-15

## Immediate (after upgrade to v2026.5.12)

### 1. `openclaw cron get <id>` for debugging
- **What**: Inspect individual cron job state by ID
- **Why**: We have work-loop and canvas-loop in error state. Currently no easy way to inspect a single job's details.
- **How**: `openclaw cron get <cron-id>` — available in v2026.5.12

### 2. `/context map` for session debugging
- **What**: Visualize context budget as a treemap
- **Why**: Our daily-review sessions hit 90k+ tokens. Understanding what consumes context helps optimize prompts.
- **How**: Run `/context map` in any session

### 3. ACP fallbacks (`acp.fallbacks`)
- **What**: Configure backup runtime backends for ACP sessions
- **Why**: Resilience when primary model is overloaded (we occasionally see model timeouts on heavy cron runs)
- **How**: Add `acp.fallbacks` to config with alternative provider entries

## Medium-term

### 4. Per-cron `timeoutSeconds`
- **What**: v2026.5.12 now honors explicit timeoutSeconds in cron payload
- **Why**: Our work-loop and study-loop can run 5-10 minutes. Currently they sometimes get cut to the default 120s.
- **How**: Add `timeoutSeconds: 600` to long-running cron payloads

### 5. `voice.allowedChannels`
- **What**: Restrict voice joins to configured channels
- **Why**: If we ever enable Discord voice, this prevents accidental joins to unwanted channels
- **How**: Not urgent, but good to know it exists
