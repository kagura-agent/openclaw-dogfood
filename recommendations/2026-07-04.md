# Adoption Recommendations — 2026-07-04

## v2026.7.1-beta.1 预览评估

Beta 发布于 07-02，包含 222 merged PRs。以下是升级到正式版后的采用优先级。

### P0 — 正式版发布后立即采用

#### 1. `on-exit` cron schedule
**场景**: 替代 subagent 完成后的 poll 等待。例如 work-loop spawn Claude Code 做 PR，用 `on-exit` 在命令完成后自动触发 review/commit 步骤。
**准备**: 梳理当前需要等待外部命令完成的 cron 场景，设计 `on-exit` 替换方案。

#### 2. Scoped conversation capability profiles
**场景**: 为 #finance channel 限制 exec 权限，为 #study channel 限制写入权限。每个对话有独立安全边界。
**准备**: 列出各 channel 的实际工具需求，设计 per-conversation profile 矩阵。

#### 3. Built-in usage footer
**场景**: 自动每轮显示 token 消耗，替代手动 /usage 检查。便于 Luna 实时了解 cost。
**准备**: 升级后确认默认启用，可能需要 config 调整关闭某些高频 channel 的 footer。

### P1 — 正式版后一周内采用

#### 4. `openclaw attach`
**场景**: 调试正在运行的 subagent session，或恢复中断的 Codex 工作流。
**准备**: 测试 attach 到 isolated cron session 的体验。

#### 5. Cursor Agent autoreview
**场景**: 扩展 code-review skill 的 reviewer 池。目前用 GPT-5.5 + Claude Opus + Gemini Pro，可以加入 Cursor Agent 做第 4 个视角。
**准备**: 评估 Cursor Agent 作为 autoreview engine 的质量 vs 成本。

### P2 — 视情况采用

#### 6. Doctor 诊断增强
下次遇到 workspace/pairing/channel-plugin 问题时用 `openclaw doctor` 排查，新增的 findings 类型覆盖面更广。

#### 7. GPT-5.6
等价格和性能评测出来后再决定是否作为 fallback model 或特定场景 model。
