## 新建议

### 1. code-review-pr-tracking 空转优化
**问题**: 91 个 tracked PR 全部 merged，无新 open PR 可 review，但 cron 仍跑完整流程消耗 67K tokens。
**建议**: 在首步添加快速检查：`gh pr list --state=open --repo openclaw/openclaw --json number | jq length`。如果 open PR 数 = 0 或仅有自己的 PR，直接输出 HEARTBEAT_OK 退出。
**预计收益**: 减少 ~60K tokens/天。

### 2. Cron 健康看板统一
**问题**: 每次 dogfood 报告都要手动从 sessions_list 聚合 channel 状态、token 消耗。信息分散，比对困难。
**建议**: 创建一个 `cron-health-dashboard.md`，由 daily-audit 或本 cron 在每次运行时自动更新：
- 各 channel 最近运行状态 / token / 异常
- 连续 error 计数
- token 消耗趋势（周对比）
**收益**: 加速诊断，减少 dogfood 报告编写时间。

### 3. PR stale 自动化决策
**问题**: PR #422 stale 2 周，#96651 接近 close 线。当前只能人工判断。
**建议**: 在 github-check 中增加 stale PR 分类逻辑：
- >7 天无活动 → 自动 ping + 标记
- >14 天 → 生成"close / rebase / 重新实现"决策建议到 discord
- 已有 spec review blocker → 提取 blocker 概要附在提醒中
**依赖**: 纯 workflow 优化，无需新 OpenClaw feature。
