# 采用建议 — 2026-06-10

## 建议升级到 2026.6.5

**优先级**: 高
**理由**: 包含多个我们遇到过的痛点修复（thinking 恢复、config.patch 数组、MCP 400 错误）

### 升级步骤
```bash
export https_proxy=http://127.0.0.1:1083
npm install -g openclaw@2026.6.5
openclaw doctor
openclaw gateway restart
```

### 升级后验证
1. 确认 `openclaw --version` 显示 2026.6.5
2. 运行 `openclaw doctor` 确认 cron JSON → SQLite 迁移正常
3. 开一个 thinking mode session，模拟 Gateway restart 后恢复
4. 用 config.patch 测试数组替换是否干净

## 新功能采用建议

### 1. Parallel 搜索 provider（中优先级）
- **场景**: 作为 web_search 的备选/fallback provider
- **操作**: 获取 PARALLEL_API_KEY，在 config 中添加为备选搜索引擎
- **收益**: 搜索可用性提升，当主 provider 限流时有后备

### 2. ClawHub GitHub repo 安装（低优先级）
- **场景**: 安装社区开发中的实验性技能
- **操作**: 等有需要时再用，当前 registry 覆盖够用
- **收益**: 能更快试用社区新技能
