## 新建议

### 1. Session context budget / checkpoint for heavy loops
- **问题**: abti-loop 单 session 消耗 2.2M tokens，远超其他 job
- **功能**: OpenClaw 支持 `contextTokens` 设置（目前所有 session 设为 1M）
- **建议**: 对 work-loop 和 abti-loop 这类可能产生长 transcript 的 job，考虑：
  - 降低 `contextTokens` 到 200K 或 500K，强制 context pruning
  - 或者在 cron payload 中设置 `timeoutSeconds` 上限（目前无限制）
  - 或者将大任务拆分为多个 subagent session（已经在做，但 parent session 也会积累）
- **Priority**: Low — 功能正常，只是成本优化
- **Risk**: 过度限制 contextTokens 可能导致中途丢失关键信息

### 2. Cron failureAlert — 重申
