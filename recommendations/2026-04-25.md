# Recommendations — 2026-04-25

## Upgrade to 2026.4.23

**Priority: High | Risk: Low**

Key gains:
- `sessions_spawn` forked context — children inherit requester transcript. Improves our heavy subagent usage (coding, rebase, analysis) by reducing context re-injection overhead.
- MCP cleanup improvements — critical for our 46-cron server to prevent orphan process accumulation.
- Memory QMD repair — fixes stale collection issues in local embeddings.
- Block streaming dedup fix — may resolve occasional duplicate messages we've seen.

## Adopt: Forked Context for Subagents

**Feature**: `sessions_spawn` with forked context (new in 2026.4.23)
**Where**: github-check → rebase subagent, workloop → coding subagent
**Why**: Currently subagents start clean and we manually inject context. Forked context lets the child see the parent's conversation when relevant.
**Action**: After upgrade, test with one coding delegation. Compare context quality vs. current approach.

## Adopt: Per-call `timeoutMs` for Generation Tools

**Feature**: Optional `timeoutMs` on image/video/music/TTS tools (new in 2026.4.23)
**Where**: kagura-canvas workflows
**Why**: ComfyUI generation sometimes takes 60-90s. Default timeouts may cut them short.
**Action**: Update kagura-canvas skill to use timeoutMs for complex generations.

## Adopt: Memory `contextSize` Tuning

**Feature**: `memorySearch.local.contextSize` config (new in 2026.4.23)
**Where**: Gateway config
**Why**: We run local embeddings on a beefy server (i9-10900X, 64GB). Default 4096 may be conservative.
**Action**: Benchmark with 8192 after upgrade. Monitor memory usage.
