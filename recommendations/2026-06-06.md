# Adoption Recommendations — 2026-06-06

## 🔴 高优：升级到 v2026.6.1

**第 3 天催促。** 以下修复每天都在影响我们的运行环境：

1. Memory watcher 稳定性 — Linux watcher 重连、写入串行化
2. Cron rate-limit 自动重试
3. Plugin 故障隔离

### 升级步骤
```bash
npm i -g openclaw@2026.6.1
openclaw gateway restart
```

### 升级后验证
1. `openclaw --version` 确认 2026.6.1
2. 观察 1-2 天 memory search 稳定性
3. 观察 cron 是否还有 rate-limit 导致的跳过

## 🟡 中优：试用 Skill Workshop

v2026.6.1 包含完整的 Skill Workshop UI 和 governed review flow。当前我们的 skill 管理完全是手动 git 操作。

**建议**：
- 升级后通过 Control UI 访问 Skill Workshop tab
- 试用一个低风险 skill 的 proposal → review → apply 流程
- 如果顺畅，逐步迁移 skill 开发到 Workshop flow

## 🟢 低优：了解 Workboard 编排

Workboard 提供了多 agent 规划和 task tracking 原语。我们当前用 FlowForge + team-lead skill 实现类似功能。

**建议**：等 Skill Workshop 采用稳定后，评估 Workboard 是否能简化 team-lead 的任务追踪逻辑。

## 🟢 低优：Code Mode Namespaces

新的 scoped agent/global sessions 和 namespace tool dispatch。对我们的多 agent 架构可能有用，但需要先了解 API。

**建议**：等升级后阅读 Code Mode MCP API docs，评估是否能替代当前的 session routing。
