# Adoption Recommendations - 2026-05-12

## Immediate (on v2026.5.7 upgrade)

### 1. Use `cron list --json` status field
- **What**: Cron JSON output now includes computed `status` (disabled/running/ok/error/skipped/idle)
- **Why**: Our toolchain-health and daily-audit crons currently derive status manually. The native field would simplify monitoring.
- **How**: Update health check scripts to use `openclaw cron list --json | jq '.[].status'`

### 2. Fix Discord cross-channel message routing
- **What**: `discord:channel:<id>` targets now route correctly instead of misrouting as DM
- **Why**: Our cron delivery uses cross-channel `message(action="send")` extensively. This fix prevents silent delivery failures.
- **How**: Automatic on upgrade. No config change needed.

## Near-term (on v2026.5.10 stable)

### 3. `/context map` for cron debugging
- **What**: Visual treemap of session context contributors
- **Why**: Morning briefing hits 77k tokens. `/context map` would show what's consuming context and help optimize.
- **When**: After v2026.5.10 stable releases

### 4. Increase `agentToAgent.maxPingPongTurns`
- **What**: Now supports up to 20 (default still 5)
- **Why**: Our work-loop→subagent flows sometimes need extended back-and-forth. Could reduce "conversation too short" edge cases.
- **How**: `config set session.agentToAgent.maxPingPongTurns 10`

### 5. Monitor HN timeout fix
- **What**: Study loop's HN connectivity has failed 2+ consecutive days
- **Why**: Not an OpenClaw feature gap — needs alternative signal source in FlowForge study workflow
- **How**: Add fallback to lobste.rs or dev.to RSS in study-loop workflow config

## Backlog

- **Why**: Subagents spawned by work-loop could be scoped to prevent accidental cross-channel messages
