# Adoption Recommendations — 2026-05-18

## 建议采用（升级后）

### 1. `/context map` — 上下文可视化 (v2026.5.12)
- **场景**: morning-briefing cron（77k tokens）和 daily-audit（74k tokens）是 token 消耗最大的 session
- **建议**: 在这些 cron 里加 `/context map` 观察 context 组成，找出可压缩部分
- **优先级**: 中

### 2. `cron.get <id>` — 单任务查看 (v2026.5.12)
- **场景**: email-dev cron error 排查时需要看单个 cron 配置
- **建议**: 替代当前的 `cron list | grep` 模式
- **优先级**: 低（等升级）

### 3. `tools.message.crossContext` — 消息工具跨上下文控制 (v2026.5.12)
- **场景**: 防止 cron session 意外向用户 channel 发消息
- **建议**: 给 cron-only agent 设置 `crossContext: false`
- **优先级**: 低

### 4. Per-sender tool policies (v2026.5.12)
- **场景**: Discord 社区频道如果将来开放给更多人
- **建议**: 暂不采用，但记录为安全加固选项
- **优先级**: 将来
