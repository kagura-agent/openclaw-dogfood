# Adoption Recommendations — 2026-06-08

## 1. Skill Workshop 正式采用 ⭐

**功能**: Governed skill proposal review flow — 提案、审批、版本管理

**现状**: 我们已有 15+ 个手动管理的 skill，通过 git 直接编辑 SKILL.md。Skill Workshop 提供结构化的提案→审批→应用流程。

**建议**: 
- 将下一次 skill 创建/修改通过 Skill Workshop 执行
- 先用在低风险场景（如 agent-memes 更新）验证流程
- 逐步迁移所有 skill 变更到 Workshop 流程

**优先级**: 🟡 Medium — 当前 git 直接管理可用，但 Workshop 提供更好的治理

## 2. Workboard 编排原语

**功能**: 多 agent 规划和任务追踪

**现状**: 我们用 pulse-todo + flowforge + 手动协调管理多 agent 任务。Workboard 提供原生的编排支持。

**建议**: 
- 了解 Workboard 的具体 API 和限制
- 评估能否替代部分 flowforge 功能
- 适合场景：code-review 多 reviewer 协调、cove-patrol + fix 子代理链路

**优先级**: 🟢 Low — 当前方案可用，待 Workboard 文档完善后评估

## 3. nightly-backup 切换压缩策略

**功能**: 不是 OpenClaw 功能，但利用 OpenClaw cron + subagent 的能力

**现状**: Teleport 和 raw tar 都 OOM，需要切换到流式压缩（pigz 管道或分卷）

**建议**:
- 修改 nightly-backup cron 的 payload，使用 `tar cf - | pigz > backup.tar.gz` 管道
- 或分卷备份避免单次内存峰值

**优先级**: 🔴 High — 备份连续失败是数据安全风险
