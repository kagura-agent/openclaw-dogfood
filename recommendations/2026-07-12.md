# Adoption Recommendations — 2026-07-12

## 基于 Beta 2026.7.1-beta.5 的采用建议

### 1. ClawRouter — 统一 Provider 管理 (优先级: 高)
**现状**: 我们使用多个 provider（Anthropic, OpenAI via floway-sg proxy），API key 分散管理，无用量预算控制。
**建议**: 2026.7.1 stable 发布后，评估 ClawRouter 作为统一 provider layer。credential-scoped 动态模型发现可以自动管理模型列表，预算报告可以控制 token 开销。
**时机**: 等 stable release

### 2. Session Titles — 改善 Session 管理 (优先级: 高)
**现状**: sessions_list 输出的 session 没有语义标题，纯靠 channel 名和 cron 名辨识。30+ 个活跃 session 时阅读困难。
**建议**: 升级后利用 generated session titles 和 utilityModel routing 自动命名 session。配合 session groups 整理 cron / channel / interactive 分类。
**时机**: 等 stable release

### 3. `openclaw attach` — 增强 ACP 工作流 (优先级: 中)
**现状**: ACP 通过 sessions_spawn 启动 Claude Code / Codex，但无法让外部 harness 连接到已有 Gateway session。
**建议**: `openclaw attach` 可以让 Claude Code 直接对已有 session 操作，带 scoped MCP grants + TTL 自动回收。适合需要人工介入的复杂调试场景。
**时机**: 等 stable release + 测试 MCP grant 行为

### 4. Cron Model Selection UI (优先级: 低)
**现状**: cron job model 通过 config 或 cron API 设置，Control UI 不直观。
**建议**: 升级后可在 Control UI 直接为每个 cron 选择模型，便于快速调整（如将低优先级 cron 切换到更便宜模型）。
**时机**: 等 stable release，低优先级

### 5. Gateway Crash-Loop Recovery (优先级: 中)
**现状**: 如果 Gateway 反复崩溃，systemd 会持续重启，可能造成日志刷屏。
**建议**: 2026.7.1 新增 crash-loop 检测 + control-plane-safe mode，反复不干净启动后自动进入安全模式。对我们的 24/7 运行至关重要。
**时机**: 随 stable release 自动获得

## 暂不采用

- **Logbook**: 需要 paired node 截屏，我们的 kagura-server 是 headless
- **Telegram Codex**: 我们不使用 Telegram channel
- **Offline mobile chat**: 对 Luna 有用但非运维优先级
- **Conversational onboarding**: 已有完整配置，不需要
