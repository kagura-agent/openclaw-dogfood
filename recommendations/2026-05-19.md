# Adoption Recommendations — 2026-05-19

## 立即采用

### 1. defineToolPlugin SDK
- **场景**: 我们有自定义 tool（如 kagura-canvas image gen）目前是手写 manifest
- **建议**: 用 `openclaw plugins init` + `defineToolPlugin` 重写，获得类型安全和 manifest 自动生成
- **effort**: 低，每个 plugin 约 30 分钟迁移

### 2. Browser dialog handling
- **场景**: browser-automation skill 在遇到 alert/confirm 时需要处理
- **建议**: 更新 browser-automation skill 使用 `browser dialog --dialog-id`
- **effort**: 低，skill 文件更新即可

### 3. Presentation rich messages
- **场景**: Discord channel 里的 poll、interactive menu、按钮选择
- **建议**: 在 community-ops 和 channel-patrol 中试用 presentation blocks
- **effort**: 中，需要了解 block schema 和各 channel 的降级行为

## 观察验证

### 4. Memory incremental sync 效果
- **观察点**: 下次 gateway restart 时检查 memory index 启动耗时
- **预期**: 应该比之前快很多，特别是 transcript 文件多的情况

### 5. Subagent completion reliability
- **观察点**: 接下来一周的 work-loop 和 github-check subagent 是否还有结果丢失
- **预期**: 多个修复应该大幅改善，但需要实际验证
