# 采用建议 — 2026-06-12

## 观察 & 建议

### 1. 考虑 Vega (Gemini 3.1 Pro) → 替代模型
- **现状**: code-review 中 Vega unique find rate 降到 8%，reliability 93%，4/6 近期 PR 有失败
- **建议**: 评估 Gemini 3.1 Flash 或等 Gemini 3.5 作为替代。也可考虑 Grok 4 作为第三评审员
- **优先级**: 中 — 当前 Stella + Nova 已足够覆盖，Vega 是锦上添花

### 2. Cron failureAlert 配置
- **现状**: 12+ 个活跃 cron，但多数没有配置 failureAlert
- **建议**: 给关键 cron（work-loop, channel-patrol, cove-patrol）配置 failureAlert.after=2，投递到 #kagura-dm
- **优先级**: 低 — 目前稳定性很好，但防患于未然

### 3. Session key 清理
- **现状**: sessions_list 显示大量已完成 session，部分 subagent session 累积
- **建议**: 定期清理或利用 OpenClaw 的 session TTL 机制（如有）减少噪音
- **优先级**: 低

### 4. lightContext 更广泛启用
- **现状**: 只有部分 cron 启用 lightContext
- **建议**: 对不需要完整 system prompt 的 cron（如 finance, auto-trader, community-ops）启用 lightContext 以节省 tokens
- **优先级**: 低 — 节省量不大但养成好习惯
