# Adoption Recommendations — 2026-07-22

## 等待 2026.7.2 Stable 的采用计划

### 🔴 高优先级（发布后立即采用）

#### 1. Cron History API
- **功能**: 从 task ledger 查看 cron 运行历史
- **场景**: 目前调试 cron 失败需要翻 session logs，有了 history API 可以直接查看运行记录
- **准备**: beta 已有，等 stable 后在 version-check job 中试用

#### 2. Skill Workshop History Review
- **功能**: 自动扫描历史 session 发现 skill ideas
- **场景**: 我们的 study-loop 和 work-loop 经常产生可复用模式，自动发现比手动创建更高效
- **准备**: 等 stable 后启用，配合现有 skill_workshop 工具

#### 3. Cron Lifecycle Conflict Retries
- **功能**: 修复 post-execution claim conflicts 导致的消息重放
- **场景**: 偶发的 cron error 可能与此相关（如本 job 7/20 的 error）
- **准备**: 升级即修复，无需配置

### 🟡 中优先级（观察后采用）

#### 4. MCP Isolation (Session-Scoped)
- **功能**: MCP server 连接限定在请求 session 内
- **场景**: 如果将来使用 MCP 工具（如外部 API bridge），隔离更安全
- **准备**: 目前未大量使用 MCP，但值得预备

#### 5. Skill Workshop Approvals 免确认
- **功能**: agent 发起的 apply/reject/quarantine 不再需要额外 approval prompt
- **场景**: work-loop 中自动创建+应用 skill 更流畅
- **准备**: 确认 `skills.workshop.approvalPolicy` 默认行为

### 🟢 低优先级（了解但暂缓）

#### 6. Remote Coding Sessions (Cloud Workers)
- **功能**: 在远端 worker 运行 Control UI sessions
- **场景**: 目前本地资源充足，但如果需要并行更多 heavy tasks 可考虑
- **准备**: 等生态成熟

#### 7. External Gateway Supervision Mode
- **功能**: `OPENCLAW_SUPERVISOR_MODE=external` 给外部生命周期管理器
- **场景**: 如果引入 OCM 或 systemd 深度集成才需要
- **准备**: 当前 systemd 管理已满足需求

## 当前可优化项（不依赖新版本）

### ✅ 已在使用且运转良好
- Isolated cron sessions with Discord delivery
- Subagent spawn with adaptive thinking
- Skill Workshop for workflow capture
- gh CLI for GitHub data access (避免 Cloudflare)

### 💡 可考虑优化
- 为 version-check job 增加 `timeoutSeconds` 避免偶发超时 error
- 考虑给 high-token jobs 设置 `contextTokens` 上限防止意外膨胀
