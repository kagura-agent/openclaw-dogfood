# 采用建议 — 2026-05-30

## 1. 升级到 v2026.5.27（紧急）

已连续 4 天报告落后。两个稳定版的累积修复量很大，特别是：
- Cron prompt-cache affinity（直接影响我们 9 个 cron job 的 token 消耗）
- Session write lock 超时释放（可能之前遇到的 session 卡死就是这个问题）
- Codex runtime 稳定性（work-loop 依赖 ACP/Codex）

**建议**：本周末找时间升级，或下次 heartbeat 时让 admin 批准自动升级。

## 2. 调查 git push 失败

上次 cron run 的 diagnostics 显示 `create git commit → push git changes (agent) failed`。
需要检查：
- SSH key / deploy key 是否过期
- 网络代理是否影响 git push
- repo 是否有 branch protection 或 force push 限制

## 3. 关注 v2026.5.28 Beta

v2026.5.28 已到 beta.4，可能在周一发布稳定版。如果选择等 .28 再升级也合理，但不要再拖了。

## 4. Token 消耗优化

work-loop (107k) 和 channel-patrol (111k) 的 token 消耗较高。升级到 .27 后的 prompt-cache affinity 优化可能会有改善。如果升级后仍然高，考虑：
- 为这些 cron job 也启用 `lightContext: true`
- 拆分 channel-patrol 的巡检范围
