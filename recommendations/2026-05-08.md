# Adoption Recommendations - 2026-05-08

## Immediate (post-upgrade to 2026.5.7)

### 1. Use `cron list --json` status field in toolchain-health
**What**: Cron JSON output now includes computed `status` (disabled/running/ok/error/skipped/idle)
**Why**: Our toolchain-health cron currently has to derive cron status manually. The new field gives it for free.
**Action**: After upgrade, update toolchain-health cron to use `openclaw cron list --json` and read `.status` directly.

### 2. Configure `subagents.archiveAfterMinutes`
**What**: Completed subagent registry rows now honor `agents.defaults.subagents.archiveAfterMinutes`
**Why**: We spawn many subagents (abti-loop, github-check, work-loop). Default 5min may be too short for debugging. Setting to 30-60min would help with post-mortem analysis.
**Action**: Add `agents.defaults.subagents.archiveAfterMinutes: 30` to config.

### 3. Leverage skill cache refresh
**What**: `/new` and `sessions.reset` now clear cached skill snapshots
**Why**: We iterate on skills frequently. Previously had to restart gateway for skill changes to take effect in long-lived sessions.
**Action**: No config needed — just awareness. Stop restarting gateway for skill updates; use `/new` instead.

## Short-term (this week)
